## 2. 通用接口约定

### 2.1 通用响应结构

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | number | 是 | 业务状态码。`0` 表示成功，非 `0` 表示业务失败或异常 |
| `message` | string | 是 | 面向前端展示或调试的提示文案 |
| `data` | object/null | 是 | 业务数据。无数据时返回 `null` |
| `requestId` | string | 否 | 请求追踪编号 |

### 2.2 通用分页结构

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `list` | array | 是 | 当前页数据列表 |
| `pageNo` | number | 是 | 当前页码，从 1 开始 |
| `pageSize` | number | 是 | 每页数量 |
| `total` | number | 是 | 总条数 |
| `hasMore` | boolean | 是 | 是否还有下一页 |

### 2.3 通用请求约定

| 约定项 | 建议 |
| --- | --- |
| 登录态 | 登录后请求头携带 `Authorization: Bearer <accessToken>` |
| 写接口幂等 | 借阅、还书、积分、注销等写接口建议请求头携带 `Idempotency-Key`（UUID）。<br>后端收到后先查该 key 是否已处理：未处理则正常执行并存储结果；已处理则直接返回上次结果，不重复执行。<br>可防止网络超时重试时产生重复借阅、重复计分等问题 |
| 时间格式 | 日期时间统一使用毫秒级时间戳，例如 `1782268200000` |
| 手机号展示 | 接口直接返回脱敏展示值，例如 `186****3173`，前端不自行脱敏 |
| 图片上传 | 使用 `POST multipart/form-data`，小程序端可用 `wx.uploadFile` |
| 错误提示 | 业务失败时 `message` 返回可展示文案，例如二维码过期、积分不足、存在未还图书 |
| 门店归属 | 前端不展示、不查询、不选择门店；员工办理借阅、还书、增加积分、消耗积分时，后端通过当前员工身份推导所属门店并落库 |

## 3. 枚举定义

| 枚举 | 值 | 说明 |
| --- | --- | --- |
| `MemberLevel` | `0` | 未办理会员卡 |
| `MemberLevel` | `10` | 普通会员（红卡） |
| `MemberLevel` | `20` | 半年卡会员（银卡） |
| `MemberLevel` | `30` | 全年卡会员（金卡） |
| `MemberCardStatus` | `active` | 有效 |
| `MemberCardStatus` | `expired` | 已过期 |
| `MemberCardStatus` | `disabled` | 不可用 |
| `MemberCardStatus` | `none` | 未办理 |
| `BorrowStatus` | `borrowing` | 借阅中 |
| `BorrowStatus` | `returned` | 已还书 |
| `PointsDirection` | `add` | 增加积分 |
| `PointsDirection` | `deduct` | 消耗积分 |
| `ReturnMode` | `view` | 查看记录 |
| `ReturnMode` | `return` | 办理还书选择 |

## 6. 认证与账号接口

### 6.1 微信手机号登录

| 项目 | 内容 |
| --- | --- |
| 接口名称 | 微信手机号登录 |
| 请求方式 | POST |
| 路径建议 | `/api/mp/v1/auth/wechat-phone-login` |
| 页面 | C01 登录弹层 |
| 说明 | 前端提交微信手机号授权返回的 `code`；后端参照微信 `getPhoneNumber` 服务端接口换取手机号，再按手机号完成账号识别和登录 |

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | string | 是 | 微信手机号授权组件返回的手机号获取凭证。该字段对应微信 `getPhoneNumber` 服务端接口请求体中的 `code`，不是 `wx.login` 返回的登录凭证 |

响应 `data`：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `accessToken` | string | 是 | 访问令牌 |
| `refreshToken` | string | 否 | 刷新令牌，如后端需要 |
| `expiresIn` | number | 是 | 访问令牌有效秒数 |
| `isStaff` | boolean | 是 | 是否为员工身份，前端据此刷新为用户首页内容或员工首页内容 |
| `userId` | string | 是 | 当前账号编号 |
| `staffId` | string/null | 否 | 员工编号，非员工为 null |

| 编号 | 事项 | 当前建议 | 影响接口 |
