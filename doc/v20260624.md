## 2. 通用接口约定

### 2.1 通用响应结构（管理后台）

管理后台使用 `AjaxResult` (继承 `HashMap`)：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | number | 是 | `200` 成功，`401` 未登录，`500` 服务端错误 |
| `msg` | string | 是 | 提示文案 |
| `data` | object/null | 否 | 业务数据，通过 `AjaxResult.success(data)` 设置时返回 |
| 其他字段 | any | 否 | 通过 `result.put(key, value)` 直接放在根节点，如 `member`、`ext` |

### 2.2 通用响应结构（小程序）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | number | 是 | 业务状态码。`0` 表示成功 |
| `message` | string | 是 | 提示文案 |
| `data` | object/null | 是 | 业务数据 |
| `requestId` | string | 否 | 请求追踪编号 |

### 2.3 通用分页结构

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `rows` | array | 是 | 当前页数据列表（管理后台字段名） |
| `total` | number | 是 | 总条数 |
| `code` | number | 是 | 状态码 |

### 2.4 通用请求约定

| 约定项 | 建议 |
| --- | --- |
| 登录态 | 登录后请求头携带 `Authorization: Bearer <accessToken>` |
| 写接口幂等 | 借阅、还书、积分等写接口建议请求头携带 `Idempotency-Key`（UUID） |
| 时间格式 | 日期时间统一使用 `yyyy-MM-dd HH:mm:ss` 字符串格式 |
| 手机号展示 | 接口直接返回脱敏展示值，例如 `186****3173`，前端不自行脱敏 |
| 图片上传 | 使用 `POST multipart/form-data` |
| 错误提示 | 业务失败时 `message` 返回可展示文案 |
| 门店归属 | 前端不展示、不查询、不选择门店；后端通过当前登录用户身份推导所属门店并落库 |

## 5. 积分系统设计（已实现）

### 5.1 积分账户模型

积分采用"书城币"记账模型，遵循复式记账原则：

```
入账 (IN)  ←── 管理员发放积分
    │
    ├── 插入 xhbs_points_order     (订单表，记录操作前后积分)
    └── 插入 xhbs_points_user_into_bill_detail (入账单，待核销)
    
出账 (OT)  ←── 管理员消耗积分（待实现）
    │
    ├── 插入 xhbs_points_order     (订单表)
    └── 插入 xhbs_points_user_out_bill_detail (出账单)
    
核销        ←── 入账单与出账单的关联
    │
    └── 插入 xhbs_points_user_bill_detail (核销明细)
```

### 5.2 添加积分流程（已实现，含并发控制）

```
┌──────────────────────────────────────────────────────┐
│ @Transactional(rollbackFor = Exception.class)         │
│                                                       │
│ ① SELECT member WHERE id=? FOR UPDATE  ← 悲观行锁    │
│    若会员不存在 → 返回错误                             │
│                                                       │
│ ② 记录 oldPoints = member.current_points              │
│    newPoints = oldPoints + points                     │
│                                                       │
│ ③ 生成订单号 IN + yyyyMMddHHmmss + 6位随机数          │
│                                                       │
│ ④ INSERT xhbs_points_order                            │
│    (orgin_points=oldPoints, after_points=newPoints)    │
│                                                       │
│ ⑤ INSERT xhbs_points_user_into_bill_detail            │
│    (points=增加额, bill_status=0-初始化)               │
│                                                       │
│ ⑥ UPDATE member SET current_points=newPoints          │
│                                                       │
│ ⑦ COMMIT → 释放行锁                                   │
│    任何步骤失败 → ROLLBACK 全部回滚                     │
└──────────────────────────────────────────────────────┘
```

### 5.3 并发安全保证

```
时间线    线程A（+50分）              线程B（+30分）
──────────────────────────────────────────────────
T1       SELECT FOR UPDATE → 获得锁
T2       读到 current_points=100
T3       INSERT order               SELECT FOR UPDATE → 等待...
T4       INSERT into_bill           ⏳ 阻塞等待A释放锁
T5       UPDATE member=150          ⏳ 阻塞等待A释放锁
T6       COMMIT → 释放锁            ⏳
T7       ─                          获得锁，读到 points=150 ✅
T8       ─                          INSERT order (orgin=150,after=180)
T9       ─                          UPDATE member=180
T10      ─                          COMMIT
──────────────────────────────────────────────────
结果     orgin=100 after=150        orgin=150 after=180 ✅ 正确
```
