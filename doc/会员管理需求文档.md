## 二、会员列表页

### 2.1 顶部查询条件
| 查询字段 | 位置 | 说明 |
|----------|------|------|
| 姓名 | — | 模糊匹配 |
| 手机号 | — | 精确匹配 |
| **门店** | 手机号后面 | 下拉选择，数据源 `sys_dept` |
| 注册时间范围 | — | 起止日期，用于周期性导出 |

### 2.2 数据列表列定义

| 列名 | 数据源 | 展示规则 |
|------|--------|----------|
| 编号 | 行序号 | 倒序展示，最新创建会员在顶部 |
| 姓名 | `member.name` | 无数据展示 "—" |
| 会员编号 | `member.card_no` | 11位全局唯一卡号 |
| 手机 | `member.phone` | 会员绑定手机号 |
| 会员类型 | `card_type` 关联名称 | 年卡 / 半年卡 / 续费年卡 / 续费半年卡 |
| 开通门店 | `sys_dept.dept_name` | 会员开卡门店 |
| 有效期 | `member.valid_date` | 到期日期，无有效期展示 "—" |
| 有效期内借阅次数 | `member.borrow_count_valid` | 数字，无数据展示 "—" |
| 积分 | `member.current_points` | 当前可用积分，无数据展示 "—" |
| 备注 | `member.remark` | 批量导入时自动填充单位名称 |
| 最近操作人 | `member.last_operator` | 最后编辑/操作该会员的员工账号 |
| 操作 | 功能按钮组 | 积分、查看、编辑、删除 |

### 2.3 操作按钮功能

| 按钮 | 行为 |
|------|------|
| **积分** | 跳转积分管理模块，单独展示该会员积分流水 |
| **查看** | 弹出【会员详情弹窗】（只读模式） |
| **编辑** | 弹出【会员编辑弹窗】（可修改全部可编辑字段） |
| **删除** | 弹出【删除二次确认弹窗】 |

### 2.4 顶部批量操作

#### 2.4.1 新增会员
- 点击弹出【新增会员弹窗】，手动单条录入会员信息

#### 2.4.2 批量导入
- 弹出文件上传弹窗，支持标准 Excel 模板上传
- **导入流程**：文件解析 → 单行校验（手机号重复、卡号重复、格式校验）→ 批量入库
- **落库规则**：
  - `member` 主表生成会员记录，`source` 标记为 `excel`
  - `member_ext` 扩展表存储单行 Excel 原始 JSON 快照
  - `member_import_log` 记录导入批次（总条数、成功/失败数、文件名、操作人）
  - `member_import_detail` 存储每一行数据（行号、处理状态、失败原因、关联 member_id）
- **导入完成后弹窗展示**：成功条数、失败条数、失败行及错误原因

#### 2.4.3 批量导出
- 导出**当前筛选条件下全部会员数据**，不受分页条数限制
- Excel 包含列表全量字段 + 隐藏扩展字段
- 导出文件命名：`赤峰新华书店会员数据_导出日期.xlsx`

## 三、功能模块 1：新增会员弹窗

### 3.1 弹窗标题
`新增会员`

### 3.2 录入字段

| 字段 | 组件 | 校验规则 |
|------|------|----------|
| **姓名** | 输入框 | **必填**，非空校验 |
| **电话** | 输入框 | **必填**，手机号格式校验，全局查重（已存在手机号禁止新增） |
| **会员类型** | 下拉选择 | **必填**，选项：年卡、半年卡 |
| **有效期展示** | 只读文本 | 选择会员类型后右侧自动计算展示 |
| **备注** | 输入框 | 非必填，填写单位名称；批量导入时强制回填 |

### 3.3 有效期计算（新增场景）
| 类型 | 计算规则 |
|------|----------|
| 年卡 | 当前系统日期 + 365 天 |
| 半年卡 | 当前系统日期 + 180 天 |

### 3.4 隐藏自动生成字段
| 字段 | 存储表 | 生成规则 |
|------|--------|----------|
| 会员卡号 `card_no` | member | 11位数字，门店编码开头，全局排重自动生成 |
| 首次入会日期 `join_date` | member_ext | 取创建当天日期 |
| 有效期内借阅次数 | member | 初始值 0 |
| 开通门店 `dept_id` | member | 取当前登录员工所属门店 ID |
| 开通员工账号 | member_ext | 取当前操作人姓名存入 `business_staff_name` |
| 数据来源 `source` | member | 标记为 `manual` |

### 3.5 底部按钮
| 按钮 | 行为 |
|------|------|
| **确认** | 校验→计算有效期→写入 member + member_ext →关闭弹窗→刷新列表 |
| **X（右上角）** | 关闭弹窗，放弃录入 |

## 四、功能模块 2：会员详情弹窗（查看按钮触发）

### 4.1 弹窗标题
`会员详情`

### 4.2 基础信息（只读）
- 姓名、电话
- 会员类型 + 到期有效期
- 会员卡号 `card_no`
- 开通门店
- 备注

### 4.3 积分获取历史表格
- 数据源：`points_history` 表
- 表格列：时间、积分变动值、操作设备（PC/小程序/POS）、操作人
- 按时间倒序展示该会员全部积分流水

### 4.4 底部按钮
| 按钮 | 行为 |
|------|------|
| **编辑** | 切换至编辑弹窗，携带当前会员所有数据 |
| **X** | 关闭详情弹窗，返回列表 |

## 五、功能模块 3：编辑会员弹窗

### 5.1 弹窗标题
`会员详情（编辑态）`

### 5.2 可编辑字段
| 字段 | 说明 |
|------|------|
| 姓名 | 可修改 |
| 电话 | 可修改，校验全局唯一 |
| 会员类型 | 下拉选择：年卡、半年卡、续费年卡、续费半年卡 |
| 备注 | 可修改 |

### 5.3 续费核心业务规则
> 会员当前卡**未到期**，选择续费类型时，有效期**叠加原有到期时间**，而非从当前日期重新计算。

| 类型 | 计算规则 |
|------|----------|
| **续费年卡** | 原 `valid_date` + 365 天 |
| **续费半年卡** | 原 `valid_date` + 180 天 |
| **普通年卡/半年卡（换卡）** | 按当前日期重新计算有效期 |

### 5.4 只读展示区
积分获取历史流水表格（同详情页，仅展示不可编辑）

### 5.5 底部按钮
| 按钮 | 行为 |
|------|------|
| **确认** | 校验→计算有效期→更新 member + member_ext →更新 `last_operator` + `updated_at` →关闭弹窗→刷新列表 |
| **X** | 放弃修改，关闭弹窗 |

## 六、功能模块 4：删除会员二次确认

### 6.1 触发条件
列表操作栏点击红色【删除】按钮

### 6.2 弹窗
| 元素 | 内容 |
|------|------|
| 文案 | "删除不可恢复，是否删除？" |
| 【确认删除】（红色） | 执行逻辑删除（`status` 置为 1 注销），刷新列表 |
| 【取消】（蓝色） | 关闭弹窗 |
| X（右上角） | 等同于取消 |

### 6.3 全局约束
系统后台**所有删除操作必须弹出二次确认弹窗**，禁止无确认直接删除数据。

## 七、会员卡号生成规则

| 规则项 | 说明 |
|--------|------|
| 长度 | **11位纯数字** |
| 编码结构 | 前 N 位 = 门店 `erp_dept_id`，后剩余为门店内自增序列 |
| 示例 | 门店 650 → `65000000001` |
| 唯一性 | 全局唯一，生成时排重，重复则取下一序列 |
| 生成时机 | 新增会员（手动/批量导入）后端自动生成，前端不可手动修改 |

## 十一、通用约束与校验规则

| 规则 | 说明 |
|------|------|
| 手机号全局唯一 | 新增/编辑/导入均校验，同一手机号不可存在多条会员记录 |
| 会员卡号全局唯一 | 后端自动生成11位，排重校验 |
| 删除数据不可逆 | 逻辑删除（status=1 注销）保留数据，弹窗强提醒 |
| 操作留痕 | 新增/编辑/导入/删除全部记录操作人+操作时间 |
| 批量导入强制回填单位 | Excel中单位字段自动写入 `member.remark` |
| 有效期自动计算 | 新增/续费场景无需人工输入，后端根据卡类型自动运算 |
