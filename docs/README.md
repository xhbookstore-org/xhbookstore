## 当前重点业务

1. 会员卡售卖：员工扫码会员动态码后开卡，后台生成售卡订单、会员卡记录、业务日志。
2. 会员卡队列：同一会员可购买多张卡，同一时间只允许一张生效；当前卡到期后按付款时间顺序激活下一张。
3. 退卡：以付款时间为准 7 天内可退，不按生效时间；退卡由 admin 后台员工操作，小程序端不提供退卡入口。
4. 会员导入：Excel 导入结果按成功、未导入、失败分组落库；卡号/会员编号已存在算未导入；系统不存在的卡类型算失败。
5. 对账闭环：售卡订单、退卡订单、会员卡、业务日志、导入日志都必须可追溯。

## 文档维护规则

- 业务规则优先写在 `业务流程`，表结构写在 `数据结构`，接口入参出参写在 `接口需求` 或 `接口文档`。
- 页面级交互只写在 `页面级需求`，避免把页面按钮逻辑散落到 SQL 或接口文档里。
- 涉及资金、会员卡状态、导入结果的变更，必须同步更新流程、表结构、接口和测试关注点。
- `doc/` 目录保留历史附件和临时资料，正式维护文档统一放在 `docs/`。
