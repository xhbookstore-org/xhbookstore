# 借阅相关表

> 版本：V2.0
>
> 状态：已开发，待执行迁移并部署
> 说明：文件名为历史兼容保留。借阅模块不维护 `book_info`、`book_info_history`、图书导入和库存。

## 1. 数据边界

- `book_info` 和 book history 不属于本期借阅链路。
- 新建借阅明细不依赖 `book_id`，只保存员工录入的 `book_code` 和 `book_name` 快照。
- 旧数据中的 `book_id` 可作为可空兼容字段保留，但新业务不再查询或写入图书主表。

## 2. `book_borrow_order` 借阅订单

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint PK | 主键 |
| `order_no` | varchar(50) UNIQUE | 借阅单号 |
| `member_id` | int | 会员 ID |
| `member_card_no/member_name/member_phone` | varchar | 借阅发生时会员快照 |
| `member_card_type_name` | varchar(100) | 会员类型快照 |
| `member_valid_date` | date | 会员有效期快照 |
| `total_book_count` | int | 逐册明细数，由服务端计算 |
| `borrow_status` | tinyint | 1借阅中/2部分处理/3已完结/4已取消 |
| `borrow_time` | datetime | 借阅时间 |
| `return_all_time` | datetime | 订单全部处理完成时间；Admin 的“最后还书时间”由归还明细聚合 |
| `dept_id` | bigint | 办理门店 |
| `first_staff_id/name` | varchar | 创建员工快照 |
| `last_staff_id/name` | varchar | 最近操作员工快照 |
| `remark` | varchar(500) | 订单备注 |
| `created_at/updated_at` | datetime | 创建和更新时间 |
| `is_del` | tinyint | 历史兼容软删除标识；业务撤销优先用状态 |

## 3. `book_borrow_detail` 逐册明细

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint PK | 逐册明细 ID |
| `borrow_order_id` | bigint | 借阅订单 ID |
| `borrow_order_no` | varchar(50) | 借阅单号快照 |
| `member_id` | int | 会员 ID |
| `book_code` | varchar(64) | 员工录入的图书编号，必填，保留前导 0 |
| `book_name` | varchar(200) | 员工录入的图书名称，必填 |
| `book_id` | bigint NULL | 仅历史数据兼容，新业务不写入、不做外键依赖 |
| `borrow_qty` | tinyint | 固定为 1，建议 CHECK 约束 |
| `returned_qty` | tinyint | 0/1 |
| `purchase_qty` | tinyint | 0/1 |
| `lost_qty` | tinyint | 0/1 |
| `borrow_status` | tinyint | 1借阅中/2已归还/5已转购/6已遗失/7已取消 |
| `borrow_time` | datetime | 借出时间 |
| `return_time` | datetime | 实物归还时间 |
| `processed_time` | datetime | 进入终态的时间 |
| `last_staff_id/name` | varchar | 最后操作员工快照 |
| `remark` | varchar(500) | 当前册备注 |
| `created_at/updated_at` | datetime | 创建和更新时间 |

单册闭环：

```text
1 = returned_qty + purchase_qty + lost_qty + cancelled_qty + outstanding_qty
```

如不增加 `cancelled_qty`，则由 `borrow_status=7` 派生撤销量，不同时使用两种口径。

## 4. `book_borrow_detail_image` 借阅明细图片

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint PK | 主键 |
| `borrow_detail_id` | bigint | 必填，关联逐册明细 |
| `image_id` | varchar(64) UNIQUE | 图片业务 ID |
| `object_key` | varchar(500) | 对象存储 Key/相对路径 |
| `thumb_object_key` | varchar(500) | 缩略图 Key |
| `image_type` | tinyint | 1借书/2归还/3损坏或其他处理凭证 |
| `sort_order` | tinyint | 1~3 |
| `image_status` | tinyint | 0正常/1已删除 |
| `file_size/width/height/format` | 数值/字符串 | 服务端校验与审计信息 |
| `operator_id/name` | varchar | 上传或最近操作人 |
| `created_at/updated_at` | datetime | 时间 |

约束：同一 `borrow_detail_id` 的有效图片总数不得超过 3。该约束必须由服务端在事务/锁内校验，不能只靠前端。

## 5. 处理记录

### 5.1 `book_return_detail`

| 字段 | 说明 |
|---|---|
| `return_order_no` | 归还单号 |
| `borrow_detail_id` | 逐册明细 ID |
| `book_code/book_name` | 从借阅明细复制的快照 |
| `return_qty` | 固定为 1 |
| `return_type` | 1正常/2损坏 |
| `return_time` | 归还时间 |
| `operator/dept/remark` | 操作员工、门店和备注 |

遗失不再伪装成“归还类型”，应使用明细状态及专用处理记录/业务日志。

### 5.2 `dd_book_purchase_order`

| 字段 | 说明 |
|---|---|
| `order_no` | 借转购订单号 |
| `borrow_detail_id` | 逐册明细 ID |
| `book_code/book_name` | 从借阅明细复制的快照 |
| `qty` | 固定为 1 |
| `receivable_amount/paid_amount` | 应收和实收 |
| `order_status` | 待支付/已支付/已取消/已退款 |
| `operator/dept/time/remark` | 操作和审计信息 |

## 6. `book_borrow_log` 业务日志

日志至少保存：

- `trace_id`、`borrow_order_id/no`、`borrow_detail_id`。
- 操作类型：创建借阅、归还、遗失、转购、取消、图片变更。
- 操作前后状态、备注和关联业务单号。
- 操作人 ID/姓名/类型、操作门店、操作端和操作时间。

借阅日志只记录借阅业务变化，不写入 book history。

## 7. 推荐索引与约束

```text
book_borrow_order:
  UNIQUE(order_no)
  INDEX(member_id, borrow_time)
  INDEX(dept_id, borrow_time)
  INDEX(borrow_status, updated_at)

book_borrow_detail:
  INDEX(borrow_order_id, borrow_status)
  INDEX(book_code)
  INDEX(book_name)

book_borrow_detail_image:
  UNIQUE(image_id)
  INDEX(borrow_detail_id, image_status, sort_order)
```

图片 3 张上限、单册终态唯一性和订单汇总状态必须在业务事务中再次校验。
