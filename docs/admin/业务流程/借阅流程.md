# 借阅流程

> 版本：V2.0
>
> 状态：目标流程，待开发
> 原则：借阅模块不访问 `book_info`、book history 和库存；每册书一条明细，每条最多 3 张图片。

## 1. 借书流程

```mermaid
flowchart TD
    A["员工扫描会员动态码"] --> B["校验会员与借阅资格"]
    B --> C{"是否可借"}
    C -- "否" --> D["拒绝并返回原因"]
    C -- "是" --> E["逐册录入图书编号、书名、备注"]
    E --> F["每册上传 0~3 张图片"]
    F --> G["整体校验字段、图片和幂等键"]
    G --> H["创建借阅订单"]
    H --> I["每个数组元素创建一条逐册明细"]
    I --> J["图片绑定对应 borrow_detail_id"]
    J --> K["写借阅业务日志并提交"]
    K --> L["调用 BORROW_BOOK 积分规则"]
    L --> M["返回借阅单号、册数和积分处理结果"]
```

借阅事务中不包含图书主数据创建、匹配、库存校验、库存扣减和图书历史写入。

## 2. 积分流程

```mermaid
flowchart TD
    A["借阅订单与明细已成功"] --> B["传入 BORROW_BOOK、借阅单号、明细数 N"]
    B --> C["查询生效的 xhbs_points_rule"]
    C --> D{"是否命中可执行规则"}
    D -- "否" --> E["记录未发放原因，借阅不回滚"]
    D -- "是" --> F["校验有效期、门店、预算、上限和幂等键"]
    F --> G["计算 N × 每册积分"]
    G --> H["写积分订单、入账明细和会员可用余额"]
    H --> I["保存规则和计算快照"]
```

同一借阅单的幂等键为 `BORROW_BOOK:{borrowOrderNo}`。初始规则为每册 10 分，但最终值必须来自规则表。

## 3. 归还、遗失和借转购

```mermaid
flowchart TD
    A["选择借阅中的逐册明细"] --> B{"处理类型"}
    B -- "归还" --> C["写归还记录，明细改已归还"]
    B -- "遗失" --> D["写遗失信息，明细改已遗失"]
    B -- "借转购" --> E["写转购订单，明细改已转购"]
    C --> F["按全部明细汇总订单状态"]
    D --> F
    E --> F
    F --> G["写操作人、操作时间和业务日志"]
```

- 不修改 `book_info`、库存和 book history。
- 正常归还、遗失和借转购不冲正借阅奖励积分。
- 只有明确的误录撤销才能关联原积分订单冲正。
- 终态明细不得再次处理。

## 4. 状态闭环

```text
单册明细：借阅中 -> 已归还 / 已遗失 / 已转购 / 已取消

订单：
- 全部明细借阅中                       -> 借阅中
- 终态明细与借阅中明细并存           -> 部分处理
- 全部明细进入终态                     -> 已完结
- 全部明细取消或订单未生效          -> 已取消
```

状态闭环只校验订单、逐册明细和处理记录，不包含库存闭环。
