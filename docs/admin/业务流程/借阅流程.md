# 借阅流程

> 状态：业务流程已设计，代码待实现

## 一、借书流程

```
员工端小程序
    │
    ▼
扫描会员动态码
    │
    ▼
POST /staff/member-code/scan
解析会员ID
    │
    ▼
GET /staff/members/{id}/overview
查看会员概要和可借阅状态
    │
    ├─ canBorrow = false ──→ 展示不可借原因（卡过期/欠费/借阅上限等）
    │
    ▼ canBorrow = true
录入借阅图书信息
  - 图书名称（必填）
  - 借书数量（默认1）
    │
    ▼
POST /staff/members/{id}/borrows
    │
    ▼
后端事务处理：
  1. 校验会员可借状态
  2. 生成借书单号：DY + yyyyMMddHHmmss + 6位随机数
  3. INSERT book_borrow_order（借书单主表）
  4. INSERT book_borrow_detail（借书明细）
  5. UPDATE book_info（扣减可借库存）
  6. INSERT book_borrow_log（操作日志，trace_id）
  7. COMMIT
     └─ 任一步失败 → ROLLBACK 全部
    │
    ▼
返回借书单号 + 借书总数
```

### 借书状态

| 状态值 | 含义 |
|--------|------|
| 1 | 已借阅（借阅中） |
| 2 | 已全部归还 |
| 3 | 部分归还 |
| 4 | 部分借转购 |
| 5 | 全部借转购 |

## 二、还书流程

```
员工端小程序
    │
    ▼
输入借书单号 或 扫描借书单码
    │
    ▼
GET /staff/borrows/{borrowId}
查看借阅详情（借书明细列表）
    │
    ▼
选择要归还的明细 → 输入还书数量
    │
    ▼
POST /staff/borrow-returns
    │
    ▼
后端事务处理：
  1. 校验借书单是否存在
  2. 校验可还数量（returned_qty + 本次 <= borrow_qty）
  3. 生成还书单号：HS + yyyyMMddHHmmss + 6位随机数
  4. INSERT book_return_detail（还书明细）
  5. UPDATE book_borrow_detail（更新returned_qty、borrow_status）
  6. UPDATE book_info（恢复可借库存）
  7. UPDATE book_borrow_order（更新borrow_status）
  8. INSERT book_return_log（操作日志，trace_id）
  9. INSERT book_borrow_log（借书日志更新，trace_id）
  10. COMMIT
    │
    ▼
返回还书单号 + 本次还书总数
```

### 还书类型

| 类型值 | 含义 |
|--------|------|
| 1 | 正常还书 |
| 2 | 损坏还书 |
| 3 | 遗失还书 |

## 三、借转购流程

```
员工端小程序
    │
    ▼
进入借阅详情 → 选择要转购的明细
    │
    ▼
录入购书信息
  - 数量
  - 售价
  - 折扣
  - 积分抵扣
    │
    ▼
POST /staff/purchase（接口待建）
    │
    ▼
后端事务处理：
  1. 校验明细可转购数量
  2. 生成购书单号：JSDD + yyyyMMddHHmmss + 6位随机数
  3. INSERT dd_book_purchase_order（购书订单）
  4. UPDATE book_borrow_detail（更新purchase_qty、borrow_status）
  5. UPDATE book_borrow_order（更新borrow_status）
  6. INSERT book_purchase_log（购书日志，trace_id）
  7. INSERT book_borrow_log（借书日志更新，trace_id）
  8. COMMIT
    │
    ▼
返回购书单号 + 应收金额
```

## 四、数量闭环校验

核心公式：

```
borrow_qty = returned_qty + purchase_qty + 未处理量
```

三层验证：

```
① 派生字段 vs 来源表
   borrow_detail.returned_qty == SUM(return_detail.return_qty)
   borrow_detail.purchase_qty == SUM(purchase_order.qty WHERE paid)

② 状态 vs 数量
   borrow_status 必须与实际处理量一致

③ 边界约束
   returned_qty + purchase_qty <= borrow_qty
```

每日定时执行闭环校验 SQL（见 `sql/book_tables_v1.sql`）。
