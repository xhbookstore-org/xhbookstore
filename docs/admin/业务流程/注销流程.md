# 注销流程

## 一、会员注销（管理后台）

### 流程

```
会员列表 → 点击"删除"
    │
    ▼
弹出【删除二次确认弹窗】
  文案："删除不可恢复，是否删除？"
    │
    ├─ 点击"取消" / X ──→ 关闭弹窗，不执行任何操作
    │
    ▼ 点击"确认删除"
调用 DELETE /member/{id}
    │
    ▼
后端处理：
  1. UPDATE member SET status = 1 WHERE id = {id}
  2. 数据保留，不物理删除
  3. 返回成功
    │
    ▼
关闭弹窗 → 刷新列表 → Toast提示"删除成功"
```

### 注销后影响

- 会员 status = 1（注销状态）
- 在会员列表中默认不显示（查询条件默认 status = 0）
- 该会员无法进行借阅、积分等操作
- 历史数据保留（借阅记录、积分流水等不受影响）

## 二、账号注销（小程序端）

### 流程

```
用户在小程序进入"账号设置"
    │
    ▼
点击"注销账号"
    │
    ▼
调用 GET /account/cancel-eligibility
查询注销前置状态
    │
    ├─ canCancel = false
    │     ├─ 有未还图书 → 提示先归还所有图书
    │     ├─ 员工身份有效 → 提示员工账号不可注销
    │     └─ 有积分余额 → 提示积分将清零
    │
    ▼ canCancel = true
弹窗二次确认："注销后数据不可恢复，确认注销？"
    │
    ▼ 确认
调用 POST /account/cancel
    │
    ▼
后端处理：
  1. 校验注销资格
  2. 标记用户状态为注销
  3. 清除登录态
  4. 返回注销时间戳
    │
    ▼
前端清除Token → 跳转到登录页
```

---

## 三、注销前置条件

| 条件 | 说明 |
|------|------|
| 无未还图书 | hasUnreturnedBooks = false |
| 非员工身份 | isStaffActive = false |
| 积分已处理 | 积分需清零或转移（待定） |
