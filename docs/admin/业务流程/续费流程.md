# 续费与换卡流程

## 一、续费（叠加有效期）

### 适用场景

会员当前卡**未到期**，需要续费延长有效期。

### 流程

```
会员列表 → 点击"编辑"
    │
    ▼
编辑弹窗 → 修改会员类型
    │
    ▼
选择"续费年卡"或"续费半年卡"
    │
    ▼
有效期计算（后端）：
    │
    ├─ 续费年卡：
    │     原 valid_date + 365 天
    │     例如：原有效期 2026-12-31 → 新有效期 2027-12-31
    │
    └─ 续费半年卡：
          原 valid_date + 180 天
          例如：原有效期 2026-12-31 → 新有效期 2027-06-29
    │
    ▼
点击"确认" → 保存
    │
    ▼
更新 member:
  - card_type_id = 续费类型ID
  - valid_date = 叠加后的日期
  - last_operator = 当前员工
  - updated_at = 当前时间
```

---

## 二、换卡（重新计算有效期）

### 适用场景

会员需要更换为不同类型的卡（如半年卡换成年卡），或原卡已到期重新开卡。

### 流程

```
会员列表 → 点击"编辑"
    │
    ▼
编辑弹窗 → 修改会员类型
    │
    ▼
选择"年卡"或"半年卡"
（非续费类型，is_renewal = 0）
    │
    ▼
有效期计算（后端）：
    │
    ├─ 年卡：
    │     当前日期 + 365 天
    │     例如：当前 2026-06-30 → 新有效期 2027-06-30
    │
    └─ 半年卡：
          当前日期 + 180 天
          例如：当前 2026-06-30 → 新有效期 2026-12-27
    │
    ▼
点击"确认" → 保存
```

---

## 三、卡类型对照

| 卡类型 | is_renewal | 有效期计算方式 |
|--------|------------|---------------|
| 年卡 | 0（新卡） | 当前日期 + 365天 |
| 半年卡 | 0（新卡） | 当前日期 + 180天 |
| 续费年卡 | 1（续费） | 原有效期 + 365天 |
| 续费半年卡 | 1（续费） | 原有效期 + 180天 |

---

## 四、判断逻辑伪代码

```
function 计算有效期(member, 新卡类型):
    if 新卡类型.is_renewal == 1:
        // 续费：叠加原有有效期
        新有效期 = member.valid_date + 新卡类型.valid_days
    else:
        // 新卡/换卡：从当天重新计算
        新有效期 = 当前日期 + 新卡类型.valid_days
    
    return 新有效期
```

---

## 五、注意事项

1. 续费操作**不需要**会员卡未到期作为前提条件（已到期也可以续费，从原到期日开始叠加）
2. 换卡操作（选择非续费类型）会将有效期从当前日期重新计算，之前的剩余有效期会丢失
3. 操作留痕：每次续费/换卡都记录 last_operator 和 updated_at
