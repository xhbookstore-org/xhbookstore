# 积分流程

## 一、积分体系概述

积分是会员的核心权益指标，贯穿借阅、购书等活动。

| 概念 | 说明 |
|------|------|
| 当前积分 | member.current_points，用户可用的积分余额 |
| 累计积分 | member_ext.total_points，历史累计获得的积分总数 |
| 积分入账 | 增加积分，生成 IN 开头的订单号 |
| 积分消耗 | 扣减积分，生成 OT 开头的订单号 |
| 积分核销 | 消耗积分时与入账单的关联关系 |

## 二、增加积分流程（管理后台）

```
会员列表 → 点击"积分"
    │
    ▼
跳转积分管理（展示该会员积分流水）
    │
    ▼
点击"增加积分"
    │
    ▼
弹出积分增加弹窗
  - 积分值（必填，正整数）
  - 积分事由（必填）
    │
    ▼
点击"确认"
    │
    ▼
调用 POST /member/{id}/points
    │
    ▼
后端事务处理（悲观锁 + 事务）：
  1. SELECT member FOR UPDATE（锁定该会员记录）
  2. 记录操作前积分：before_points = current_points
  3. 计算操作后积分：after_points = before_points + points
  4. 生成订单号：IN + yyyyMMddHHmmss + 随机数
  5. INSERT xhbs_points_order（积分订单表）
     - direction = "add"
     - before_points / after_points
  6. INSERT xhbs_points_user_into_bill_detail（入账单明细）
  7. UPDATE member SET current_points = after_points
  8. COMMIT
    │
    │  并发场景：同一会员并发加积分
    │  → 后者等待前者提交后才能读取最新积分
    │  → 悲观锁保证数据一致性
    │
    ▼
返回成功 → Toast提示 → 刷新积分流水列表
```

## 三、消耗积分流程（API端）

```
员工端小程序
    │
    ▼
查看会员概要 → canDeductPoints = true
    │
    ▼
选择消耗积分事项
    │
    ▼
输入消耗积分值（正整数，不超过当前积分）
    │
    ▼
POST /staff/members/{id}/points/deduct
    │
    ▼
后端处理：
  1. 校验积分余额充足（消耗值 <= current_points）
  2. 校验积分事项有效
  3. 记录 before_points / after_points
  4. INSERT xhbs_points_order（direction = "deduct"）
  5. INSERT xhbs_points_user_out_bill_detail（出账单明细）
  6. UPDATE member.current_points
  7. COMMIT
    │
    ▼
返回成功 + 积分变动详情
```

## 四、积分流水查询

### 管理后台

```
会员列表 → 点击"积分" → 展示该会员积分流水
```

调用 `GET /member/{id}/points`，返回该会员所有积分订单记录。

### 小程序端（用户）

```
我的 → 积分记录
```

调用 `GET /user/points-records`，返回当前用户积分记录。

### 小程序端（员工）

```
员工首页 → 查看会员 → 积分记录
```

调用 `GET /staff/points-records?memberId=xxx`。

## 五、积分并发安全

```
场景：员工A和员工B同时对会员X增加积分

时间线：
  T1: 员工A SELECT member WHERE id=X FOR UPDATE（获取锁）
  T2: 员工B SELECT member WHERE id=X FOR UPDATE（等待A释放锁）
  T3: 员工A INSERT order + UPDATE member + COMMIT（释放锁）
  T4: 员工B 获取锁，读取最新 current_points
  T5: 员工B INSERT order + UPDATE member + COMMIT

结果：两个操作串行执行，积分数据一致
```

## 六、订单号规则

| 方向 | 前缀 | 格式 | 示例 |
|------|------|------|------|
| 入账 | IN | IN + yyyyMMddHHmmss + 6位随机数 | IN20260630120000123456 |
| 出账 | OT | OT + yyyyMMddHHmmss + 6位随机数 | OT20260630120000654321 |

## 七、积分事项

积分事项用于标识积分变动的原因。

### 增加积分事项

| 事项 | 说明 |
|------|------|
| 活动赠送 | 营销活动赠送积分 |
| 借阅奖励 | 借书行为奖励积分 |
| 购书返利 | 购书消费返积分 |
| 手动调整 | 管理员手动调整 |

### 消耗积分事项

| 事项 | 说明 |
|------|------|
| 兑换礼品 | 积分兑换实物礼品 |
| 抵扣书款 | 购书时积分抵扣 |
| 过期清零 | 年度积分过期清零 |
