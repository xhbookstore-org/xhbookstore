# 会员管理流程

## 一、新增会员流程

```
员工登录管理后台
    │
    ▼
进入"会员列表"页面
    │
    ▼
点击"新增"按钮
    │
    ▼
弹出【新增会员弹窗】
    │
    ▼
填写会员信息：
  - 姓名（必填）
  - 电话（必填，全局查重）
  - 会员类型（必填，下拉选择：年卡/半年卡）
  - 备注（选填）
    │
    ▼
选择会员类型后
    │  ┌─ 年卡 → 有效期 = 当前日期 + 365天
    │  └─ 半年卡 → 有效期 = 当前日期 + 180天
    │
    ▼
点击"确认"
    │
    ▼
前端校验
    │  ├─ 必填字段非空
    │  └─ 手机号格式
    │
  ┌─ 校验失败 ──→ 字段下方红色提示
  │
  ▼ 校验通过
调用 POST /member
    │
    ▼
后端处理：
  1. 校验手机号全局唯一 → 已存在则返回错误
  2. 生成11位会员卡号（门店编码 + 自增序列）
  3. 取当前员工门店作为开通门店
  4. 计算有效期
  5. 写入 member 表（source = "manual"）
  6. 写入 member_ext 表（join_date、business_staff_name）
  7. 返回成功
    │
  ┌─ 失败 ──→ Toast提示错误信息
  │
  ▼ 成功
关闭弹窗 → 刷新会员列表 → Toast提示"新增成功"
```

## 二、编辑会员流程

```
会员列表 → 点击"编辑"
    │
    ▼
弹出【编辑会员弹窗】
加载会员当前数据 + 积分历史流水
    │
    ▼
修改可编辑字段：
  - 姓名
  - 电话（全局唯一校验）
  - 会员类型
  - 备注
    │
    ▼
会员类型变更 → 有效期重新计算
  │
  ├─ 选择"续费年卡/续费半年卡"
  │     └─ 有效期 = 原valid_date + 对应天数
  │     └─ 适用场景：会员当前卡未到期
  │
  └─ 选择"年卡/半年卡"（换卡）
        └─ 有效期 = 当前日期 + 对应天数
        └─ 适用场景：会员卡类型完全变更
    │
    ▼
点击"确认"
    │
    ▼
前端校验 → 调用 PUT /member/{id}
    │
    ▼
后端处理：
  1. 校验手机号唯一（排除自身）
  2. 根据卡类型计算有效期
  3. 更新 member + member_ext
  4. 更新 last_operator + updated_at
  5. 返回成功
    │
    ▼
关闭弹窗 → 刷新列表 → Toast提示"修改成功"
```

## 三、删除会员流程

```
会员列表 → 点击"删除"
    │
    ▼
弹出【删除二次确认弹窗】
  文案："删除不可恢复，是否删除？"
  按钮：[确认删除]（红色） [取消]（默认）
    │
    ├─ 点击"取消" / X ──→ 关闭弹窗
    │
    ▼ 点击"确认删除"
调用 DELETE /member/{id}
    │
    ▼
后端处理：
  1. 逻辑删除，status 置为 1（注销）
  2. 数据不物理删除
  3. 返回成功
    │
  ┌─ 失败 ──→ Toast提示"删除失败"
  │
  ▼ 成功
关闭弹窗 → 刷新列表 → Toast提示"删除成功"
```

## 四、批量导入流程

```
会员列表 → 点击"批量导入"
    │
    ▼
弹出【导入弹窗】
  - 下载Excel模板
  - 选择文件上传
    │
    ▼
点击"上传"
    │
    ▼
调用 POST /member/import
上传Excel文件
    │
    ▼
后端处理：
  1. 解析Excel文件
  2. 逐行校验：
     - 手机号重复（与库内已有数据比对）
     - 卡号重复
     - 必填字段校验
     - 格式校验（手机号、日期等）
  3. 事务批量入库：
     - member 表写入（source = "excel"）
     - member_ext 表写入（excel_raw_data 存储原始JSON）
     - member_import_log 写入批次日志
     - member_import_detail 写入每行处理结果
  4. 返回导入结果
    │
    ▼
弹窗展示导入结果：
  - 总条数：100
  - 成功条数：95
  - 失败条数：5
  - 失败行详情：
    │ 行号 │ 姓名 │ 失败原因 │
    │  3   │ 张三 │ 手机号重复 │
    │  7   │ 李四 │ 姓名不能为空 │
    │  ...
    │
    ▼
关闭弹窗 → 刷新列表
```

## 五、批量导出流程

```
会员列表
    │
    ▼
设置筛选条件（可选）
  - 姓名、手机号、门店、注册时间范围
    │
    ▼
点击"批量导出"
    │
    ▼
调用 GET /member/export（携带筛选参数）
    │
    ▼
后端处理：
  1. 根据筛选条件查询全部数据（不分页）
  2. 生成Excel文件
  3. 返回文件流下载
    │
    ▼
浏览器下载Excel文件
文件命名：赤峰新华书店会员数据_YYYYMMDD.xlsx
```

## 六、会员卡号生成规则

```
新增会员时触发
    │
    ▼
调用 GET /member/generateCardNo?deptId=xxx
    │
    ▼
后端规则：
  1. 获取门店编码（erp_dept_id 或 dept_id）
  2. 查询该门店下已有的最大卡号
  3. 最大卡号 + 1
  4. 补零到11位
  5. 全局查重 → 重复则继续 +1
  6. 返回卡号
    │
    ▼
示例：门店 650 → "65000000001"
```
