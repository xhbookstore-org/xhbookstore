# 权限设计

## 概述

管理后台采用 RBAC（基于角色的访问控制）权限模型，通过 用户 → 角色 → 菜单/权限 的三层关系实现权限控制。

---

## 一、权限模型

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│  用户     │────→│  用户-角色    │←────│   角色        │
│ sys_user │  N:M │ user_role    │ M:N │  sys_role    │
└──────────┘     └──────────────┘     └──────────────┘
                                             │
                                             │ M:N
                                             ▼
                 ┌──────────────┐     ┌──────────────┐
                 │  角色-菜单    │     │   菜单/权限   │
                 │ role_menu    │     │  sys_menu    │
                 └──────────────┘     └──────────────┘
```

---

## 二、角色设计

### 预置角色

| 角色 | 角色标识 | 数据范围 | 说明 |
|------|----------|----------|------|
| 超级管理员 | admin | 全部数据 | 拥有系统全部权限 |
| 普通管理员 | manager | 本部门及以下 | 管理部门内数据 |
| 店员 | staff | 仅本人数据 | 日常业务操作 |
| 会员管理员 | member_admin | 本部门及以下 | 专门管理会员数据 |

### 数据权限范围

| 范围值 | 含义 | 说明 |
|--------|------|------|
| 1 | 全部数据权限 | 不受部门限制 |
| 2 | 自定义数据权限 | 指定可访问的部门 |
| 3 | 本部门数据权限 | 仅本部门 |
| 4 | 本部门及以下数据权限 | 本部门及下级部门 |
| 5 | 仅本人数据权限 | 仅自己创建的数据 |

## 三、菜单/权限设计

### 菜单类型

| 类型 | 标识 | 说明 |
|------|------|------|
| 目录 | M | 一级菜单分组，无实际页面 |
| 菜单 | C | 页面级，关联路由和Vue组件 |
| 按钮 | F | 页面内操作按钮的权限标识 |

### 权限标识命名规范

`模块:实体:操作`

| 模块 | 实体 | 操作 |
|------|------|------|
| system | user / role / menu / dept / post / dict / config / notice | list / add / edit / remove / export |
| monitor | online / job / server / cache / operlog / logininfor | list / add / edit / remove / forceLogout |
| member | member / points | list / add / edit / remove / export / import |
| tool | gen / swagger / build | list / edit / remove |

## 四、权限控制层级

### 第一层：登录认证
- 通过 SpringSecurity + Session Cookie 实现
- 登录成功后创建服务端 Session
- 未登录请求返回 302 重定向到登录页

### 第二层：路由/菜单权限
- 登录后 `/getRouters` 接口返回用户有权限的路由
- 前端动态添加路由
- 无权限的菜单在侧边栏不显示

### 第三层：页面访问权限
- 路由守卫检查用户是否有访问目标路由的权限
- 无权限跳转 401 页面

### 第四层：按钮/操作权限
- 前端：`v-hasPermi` 指令控制按钮显示
- 后端：`@PreAuthorize("@ss.hasPermi('xxx')")` 注解控制接口访问

## 五、前端实现

### 路由配置

```javascript
{
  path: '/member',
  component: Layout,
  meta: { title: '会员管理', icon: 'user' },
  children: [
    {
      path: 'list',
      component: () => import('@/views/member/list'),
      meta: { title: '会员列表' },
      permissions: ['member:member:list']  // 所需权限标识
    }
  ]
}
```

### 按钮控制

```vue
<el-button v-hasPermi="['member:member:add']">新增</el-button>
<el-button v-hasPermi="['member:member:edit']">编辑</el-button>
<el-button v-hasPermi="['member:member:remove']">删除</el-button>
```

## 六、后端实现

### 接口权限控制

```java
@PreAuthorize("@ss.hasPermi('member:member:list')")
@GetMapping("/list")
public TableDataInfo list(Member member) { ... }

@PreAuthorize("@ss.hasPermi('member:member:add')")
@PostMapping
public AjaxResult add(@RequestBody Member member) { ... }
```

### 数据权限控制
- 查询列表时，根据用户数据权限范围自动拼接 WHERE 条件
- 使用 MyBatis 拦截器或 Service 层统一处理
