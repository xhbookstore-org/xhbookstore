# 通用交互规范

> 本文档汇总管理后台所有页面共用的交互规范。各页面文档中不再重复描述，直接引用本文档。

---

## 一、Toast 消息提示

所有操作结果使用 Element UI `Message` 组件。

### 成功提示
- 操作成功后，顶部绿色提示条，3秒自动消失
- 文案格式："操作成功" / "XXX成功"

### 失败提示
- 操作失败后，顶部红色提示条
- 文案格式：接口返回的 message 或 "操作失败，请重试"

### 警告提示
- 需要用户注意时，顶部黄色提示条

---

## 二、Loading 加载

### 页面级 Loading
- 首次进入页面、切换Tab时显示
- 使用骨架屏或 Spin 组件
- 加载完成后切换为实际内容

### 按钮级 Loading
- 提交按钮点击后显示 loading 状态
- 禁止重复点击（防抖）
- 接口返回后恢复

### 列表级 Loading
- 查询/翻页时列表区域显示加载动画
- 使用 v-loading 指令

---

## 三、Dialog 弹窗

### 统一规范
- 弹窗宽度：新增/编辑 600-800px，详情 600-900px，确认 420px
- 弹窗标题：明确操作类型（新增XX / 编辑XX / XX详情 / 确认删除）
- 底部按钮：
  - 表单弹窗：确认（主色） + 取消（默认色）
  - 确认弹窗：确认删除（红色/danger） + 取消（默认色）
  - 详情弹窗：关闭（默认色） + 编辑（可选）
- 右上角 X：等同于取消
- 点击遮罩层：不关闭弹窗（`:close-on-click-modal="false"`）
- ESC键：关闭弹窗

### 关闭时重置
- 表单弹窗关闭时清空表单数据和校验状态

---

## 四、Empty 空状态

### 列表无数据
- 表格区域中间显示空状态图标 + "暂无数据"

### 搜索无结果
- 显示 "未找到匹配的数据" + "清空筛选"链接

### 下拉无数据
- 显示 "暂无数据"

---

## 五、分页组件

### 统一配置
- 默认每页 20 条
- 可选每页条数：10 / 20 / 50 / 100
- 页码按钮：最多显示 7 个页码
- 显示总条数：`共 XX 条`
- 支持快速跳转到指定页码

### 操作行为
- 切换页码 → 重新请求数据
- 切换每页条数 → 回到第1页，重新请求
- 查询/筛选 → 重置为第1页

---

## 六、删除二次确认

### 全局约束
系统后台**所有删除操作必须弹出二次确认弹窗**，禁止无确认直接删除数据。

### 弹窗内容
- 标题：`提示` / `确认删除`
- 文案：`"删除不可恢复，是否删除？"`（具体文案视场景调整）
- 确认按钮：红色/danger，文案 `"确认删除"`
- 取消按钮：默认色，文案 `"取消"`
- 右上角X：等同于取消

### 执行流程
1. 点击删除按钮
2. 弹出确认弹窗
3. 点击确认 → 调用删除接口
4. 成功后 → Toast提示 + 刷新列表
5. 失败后 → Toast提示错误，不关闭弹窗

---

## 七、权限判断

### 页面级权限
- 无权限的菜单在侧边栏不显示
- 通过路由的 `permissions` 字段控制

### 按钮级权限
- 使用 `v-hasPermi` 指令控制按钮显示/隐藏
- 无权限的按钮不可见
- 示例：`<el-button v-hasPermi="['system:user:add']">新增</el-button>`

### 角色级权限
- 使用 `v-hasRole` 指令
- 示例：`<el-button v-hasRole="['admin']">管理员操作</el-button>`

---

## 八、表单校验

### 统一规则
- 必填字段：输入框后带红色 `*` 标记
- 校验时机：提交时校验 + 失焦时校验
- 错误提示：输入框下方红色文字

### 常见校验
- 非空：`required: true, message: '请输入XXX', trigger: 'blur'`
- 手机号：正则 `/^1[3-9]\d{9}$/`
- 邮箱：type: 'email'
- 长度限制：`min` / `max`

---

## 九、网络异常处理

### 统一处理
- 接口超时：30秒
- 网络断开：Toast提示"网络连接失败，请检查网络"
- 401 未登录：自动跳转登录页
- 403 无权限：跳转401页面
- 500 服务端错误：展示接口返回的错误信息

---

## 十、操作留痕

### 自动记录
- 新增操作：记录创建人和创建时间
- 编辑操作：更新最后操作人和更新时间
- 删除操作：记录删除操作人（逻辑删除不物理删除数据）
- 导入操作：记录导入批次（总条数、成功/失败数、操作人）
