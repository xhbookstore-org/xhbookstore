# 权限指令

## 说明

Vue 自定义指令，用于控制页面元素的显示/隐藏，实现按钮级权限控制。

---

## 一、v-hasPermi — 权限标识控制

### 说明

根据用户拥有的权限标识集合，控制元素的显示。

### 使用方式

```vue
<!-- 单个权限 -->
<el-button v-hasPermi="['system:user:add']">新增</el-button>

<!-- 多个权限（OR逻辑，满足任一即可） -->
<el-button v-hasPermi="['system:user:add', 'system:user:edit']">新增/编辑</el-button>
```

### 权限标识格式

`模块:实体:操作`

| 操作 | 含义 | 示例 |
|------|------|------|
| list | 列表/查询 | `system:user:list` |
| add | 新增 | `system:user:add` |
| edit | 编辑 | `system:user:edit` |
| remove | 删除 | `system:user:remove` |
| export | 导出 | `system:user:export` |
| import | 导入 | `member:member:import` |

### 会员管理权限标识

| 权限标识 | 对应操作 |
|----------|----------|
| `member:member:list` | 会员列表查询 |
| `member:member:add` | 新增会员 |
| `member:member:edit` | 编辑会员 |
| `member:member:remove` | 删除会员 |
| `member:member:export` | 导出会员 |
| `member:member:import` | 批量导入 |
| `member:points:add` | 添加积分 |

---

## 二、v-hasRole — 角色控制

### 说明

根据用户角色控制元素的显示。

### 使用方式

```vue
<!-- 单个角色 -->
<el-button v-hasRole="['admin']">管理员操作</el-button>

<!-- 多个角色（OR逻辑） -->
<el-button v-hasRole="['admin', 'member_manager']">管理员/会员管理操作</el-button>
```

---

## 三、权限控制层级

```
用户 → 角色 → 菜单权限 + 按钮权限
                    │
                    ├── 页面级：菜单路由控制（配置在菜单管理中）
                    │     └── 无权限菜单不显示在侧边栏
                    │
                    └── 按钮级：v-hasPermi 指令控制
                          └── 无权限按钮不渲染
```

---

## 四、权限数据来源

登录成功后，后端返回用户权限集合：

```
GET /getInfo → { permissions: ["system:user:list", "system:user:add", ...], roles: ["admin"] }
```

前端存储到 Vuex store，`v-hasPermi` 和 `v-hasRole` 指令从 store 中读取。
