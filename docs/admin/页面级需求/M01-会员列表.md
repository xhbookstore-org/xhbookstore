# M01-会员列表

## 页面说明

会员管理核心页面，支持会员信息的查询、编辑、删除、导入导出，以及积分管理。新会员仅支持批量导入和小程序端注册。页面风格模拟"用户管理"页面，但去掉组织结构树。

## 页面入口

- 侧边栏菜单 → 会员管理 → 会员列表
- 路由：`/member/list`

## 页面权限

- 权限标识：`member:member:list`
- 操作权限：`member:member:add` / `member:member:edit` / `member:member:remove` / `member:member:export` / `member:member:import` / `member:points:add`

## 页面元素

- **顶部面包屑导航**：会员管理 > 会员列表
- **查询条件区**：
  - 姓名（输入框，模糊匹配）
  - 手机号（输入框，精确匹配）
  - 门店（下拉选择，数据源 sys_dept）
  - 注册时间范围（起止日期选择器）
  - 查询按钮 / 重置按钮
- **批量操作区**：批量导入 / 批量导出
- **数据列表**：
  - 序号（倒序，最新创建在顶部）
  - 姓名（无数据展示 "—"）
  - 会员编号（11位全局唯一卡号）
  - 手机
  - 会员类型（年卡/半年卡/续费年卡/续费半年卡）
  - 开通门店
  - 有效期（无有效期展示 "—"）
  - 有效期内借阅次数（无数据展示 "—"）
  - 积分（无数据展示 "—"）
  - 备注
  - 最近操作人
  - 操作按钮组（积分、查看、编辑、删除），固定在表格右侧
- **分页组件**：页码、每页条数、总条数

## 页面交互

### 查询会员
1. 输入查询条件（支持多条件组合）
2. 点击"查询"按钮
3. 调用 GET /member/list 接口
4. 刷新数据列表，页码重置为第1页

### 点击重置
- 清空所有查询条件，重新加载默认列表

### 点击批量导入
- 弹出文件上传弹窗，支持上传标准Excel模板
- 导入流程：文件解析 → 单行校验（手机号重复、卡号重复、格式校验）→ 批量入库
- 落库规则：
  - member 主表生成会员记录，source 标记为 excel
  - member_ext 扩展表存储单行 Excel 原始 JSON 快照
  - member_import_log 记录导入批次
  - member_import_detail 存储每一行处理状态
- 导入完成后弹窗展示：成功条数、失败条数、失败行及错误原因

### 点击批量导出
- 导出**当前筛选条件下全部会员数据**，不受分页条数限制
- Excel 包含列表全量字段 + 隐藏扩展字段
- 命名：`赤峰新华书店会员数据_导出日期.xlsx`

### 点击积分按钮
- 跳转积分管理模块，单独展示该会员积分流水

### 点击查看按钮
- 弹出【会员详情弹窗】（只读模式，详见下方子模块2）

### 点击编辑按钮
- 弹出【会员编辑弹窗】（可修改全部可编辑字段，详见下方子模块3）

### 点击删除按钮
- 弹出【删除二次确认弹窗】（详见下方子模块4）

### 分页操作
- 点击页码切换页面，可选择每页显示条数（10/20/50/100）

## 字段说明

| beginTime | date | 否 | — | 注册开始日期 |
| endTime | date | 否 | — | 注册结束日期 |
| pageNo | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 20 | 每页条数 |

| 字段名称 | 类型 | 数据源 | 展示规则 |
|----------|------|--------|----------|
| 序号 | int | 行号 | 倒序展示 |

| 会员编号 | string | member.card_no | 11位全局唯一卡号 |
| 手机 | string | member.phone | 会员绑定手机号 |
| 会员类型 | string | card_type关联名称 | 年卡/半年卡/续费年卡/续费半年卡 |

| 积分 | int | member.current_points | 当前可用积分，无数据展示 "—" |
| 备注 | string | member.remark | 批量导入时自动填充单位名称 |

| 最近操作人 | string | member.last_operator | 最后编辑/操作该会员的员工账号 |

## 接口

- `GET /member/list` — 会员列表（分页+条件查询）
- `GET /member/{id}` — 会员详情（含扩展表）
- `POST /member` — 管理后台不开放新增，返回引导提示；仅批量导入和小程序端注册可创建会员
- `PUT /member/{id}` — 编辑会员
- `DELETE /member/{id}` — 删除会员（逻辑删除）
- `GET /member/generateCardNo` — 生成11位会员卡号
- `GET /member/cardTypes` — 卡类型列表
- `GET /member/export` — 批量导出
- `GET /member/{id}/points` — 会员积分流水
- `POST /member/{id}/points` — 添加积分

## 数据校验

| 规则 | 说明 |
|------|------|
| 手机号全局唯一 | 新增/编辑/导入均校验，同一手机号不可存在多条会员记录 |
| 会员卡号全局唯一 | 后端自动生成11位，排重校验 |
| 管理端新增入口 | 隐藏；仅允许批量导入和小程序端注册创建会员 |
| 导入必填字段 | 会员编号、姓名、入会日期、会员类型、到期时间必须完整；任一缺失则该行不创建会员，手机号可修正后再次导入 |
| 删除数据不可逆 | 逻辑删除（status=1 注销）保留数据，弹窗强提醒 |
| 操作留痕 | 新增/编辑/导入/删除全部记录操作人+操作时间 |
| 批量导入强制回填单位 | Excel中单位字段自动写入 member.remark |
| 有效期自动计算 | 新增/续费场景无需人工输入，后端根据卡类型自动运算 |

## 异常处理

| 场景 | 处理方式 |
|------|----------|
| 列表接口失败 | Toast提示"数据加载失败"，保留上一次数据 |
| 无数据 | 列表区域展示空状态："暂无会员数据" |
| 删除失败 | Toast提示"删除失败，请重试" |
| 导入失败 | 弹窗展示失败详情（行号+失败原因） |
| 网络异常 | Toast提示"网络连接失败" |
| 权限不足 | 按钮置灰或隐藏，点击时提示"无操作权限" |

## UI状态

- **Loading**：列表区域显示加载动画
- **Empty**：显示空状态图标 + "暂无数据"
- **Error**：Toast提示错误信息
- **Success**：正常展示数据列表

## 子模块1：新增会员弹窗

### 弹窗标题
`新增会员`

### 录入字段
| 字段 | 组件 | 校验规则 |
|------|------|----------|
| **姓名** | 输入框 | **必填**，非空校验 |
| **电话** | 输入框 | **必填**，手机号格式校验，全局查重 |
| **会员类型** | 下拉选择 | **必填**，选项：年卡、半年卡 |
| **有效期展示** | 只读文本 | 选择会员类型后右侧自动计算展示 |
| **备注** | 输入框 | 非必填，填写单位名称 |

### 有效期计算（新增场景）
| 类型 | 计算规则 |
|------|----------|
| 年卡 | 当前系统日期 + 365 天 |
| 半年卡 | 当前系统日期 + 180 天 |

### 隐藏自动生成字段
| 字段 | 存储表 | 生成规则 |
|------|--------|----------|
| card_no | member | 11位数字，门店编码开头，全局排重自动生成 |
| join_date | member_ext | 取创建当天日期 |
| 有效期内借阅次数 | member | 初始值 0 |
| dept_id | member | 取当前登录员工所属门店 ID |
| business_staff_name | member_ext | 取当前操作人姓名 |
| source | member | 标记为 manual |

### 底部按钮
| 按钮 | 行为 |
|------|------|
| 确认 | 校验→计算有效期→写入member+member_ext→关闭弹窗→刷新列表 |
| X（右上角） | 关闭弹窗，放弃录入 |

| 按钮 | 行为 |

| 编辑 | 切换至编辑弹窗，携带当前会员所有数据 |

## 子模块3：编辑会员弹窗

### 弹窗标题
`会员详情（编辑态）`

### 可编辑字段
| 字段 | 说明 |
|------|------|
| 姓名 | 可修改 |
| 电话 | 可修改，校验全局唯一 |
| 会员类型 | 下拉选择：年卡、半年卡、续费年卡、续费半年卡 |
| 备注 | 可修改 |

### 续费核心业务规则
> 会员当前卡**未到期**，选择续费类型时，有效期**叠加原有到期时间**，而非从当前日期重新计算。

| 类型 | 计算规则 |
|------|----------|
| **续费年卡** | 原 valid_date + 365 天 |
| **续费半年卡** | 原 valid_date + 180 天 |
| **普通年卡/半年卡（换卡）** | 按当前日期重新计算有效期 |

### 只读展示区
积分获取历史流水表格（同详情页，仅展示不可编辑）

### 底部按钮
| 按钮 | 行为 |
|------|------|
| 确认 | 校验→计算有效期→更新member+member_ext→更新last_operator+updated_at→关闭弹窗→刷新列表 |
| X | 放弃修改，关闭弹窗 |

## 子模块4：删除确认弹窗

### 触发条件
列表操作栏点击红色"删除"按钮

### 弹窗
| 元素 | 内容 |
|------|------|
| 文案 | "删除不可恢复，是否删除？" |
| 确认删除（红色） | 执行逻辑删除（status置为1注销），刷新列表 |
| 取消（蓝色） | 关闭弹窗 |
| X（右上角） | 等同于取消 |

### 全局约束
系统后台**所有删除操作必须弹出二次确认弹窗**，禁止无确认直接删除数据。
