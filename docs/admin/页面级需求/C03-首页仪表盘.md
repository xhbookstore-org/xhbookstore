# C03-首页会员数据看板

## 页面说明

后台登录后的默认首页，用于展示当前登录人数据权限范围内的会员经营汇总数据。页面不由前端传门店参数，后端根据登录人的部门、角色数据权限自动计算可见范围。

## 页面入口

- 登录成功后进入首页。
- 侧边栏点击“首页”。

## 页面权限

- 所有已登录 admin 后台用户可访问。
- 数据范围由后端按登录人数据权限控制：
  - 超级管理员：查看全部正常部门数据。
  - 全部数据权限角色：查看全部正常部门数据。
  - 自定义数据权限角色：查看角色绑定部门数据。
  - 本部门数据权限角色：查看登录人所属部门数据。
  - 本部门及以下数据权限角色：查看登录人所属部门及子部门数据。
  - 仅本人数据权限角色：首页按登录人所属部门统计。

## 查询规则

1. 页面初始化时调用 `GET /dashboard/member/overview`。
2. 前端不传 `deptId`、时间范围、角色范围等查询条件。
3. 后端通过 `SecurityUtils.getLoginUser()` 获取当前登录人。
4. 后端根据登录人角色数据权限解析可见部门 ID 列表。
5. 首页统计数据从 Redis 读取各部门预聚合数据，并按可见部门 ID 合并返回。
6. 如果 Redis 尚未初始化，接口会触发一次带分布式锁的刷新，避免首页首次打开为空。
7. 手动点击“刷新统计”时调用 `POST /dashboard/member/refresh`，刷新成功后重新查询首页数据。
8. 如果多节点同时刷新，Redis 锁只允许一个节点执行，其余节点返回“已有其他节点正在刷新”。

## 缓存与定时任务规则

| 项目 | 规则 |
|---|---|
| 聚合方式 | 按部门预聚合后存入 Redis |
| 刷新频率 | admin 服务每 5 分钟执行一次 |
| 并发控制 | 使用 Redis 分布式锁 `lock:dashboard:member-stats:refresh` |
| 部门缓存键 | `dashboard:member:dept:{deptId}` |
| 登录统计缓存键 | `dashboard:member:login` |
| 已缓存部门列表 | `dashboard:member:dept_ids` |
| 最后刷新时间 | `dashboard:member:last_refresh_at` |
| 最后刷新状态 | `dashboard:member:last_refresh_status` |

## 字段规则

### 会员数据

| 页面字段 | 统计规则 |
|---|---|
| 累计会员 | 当前数据权限范围内 `member.status = 0` 的会员数量 |
| 借阅卡会员 | 当前数据权限范围内存在有效会员卡的会员数量；有效会员卡指 `member_card.status = 1` 且 `is_del = 0` |
| 普通会员 | 累计会员 - 借阅卡会员 |
| 单位借阅卡会员 | 有效借阅卡会员中，`member.remark` 非空的会员数量 |
| 自然借阅卡会员 | 有效借阅卡会员中，`member.remark` 为空的会员数量 |
| 可见部门 | 当前登录人根据数据权限解析出的部门数量 |

### 会员卡类型数据

| 页面字段 | 统计规则 |
|---|---|
| 尊享会员 | 有效会员卡中 `valid_days >= 365` 的会员数量 |
| 尊享会员-单位会员 | 尊享会员中 `member.remark` 非空的会员数量 |
| 尊享会员-自然会员 | 尊享会员中 `member.remark` 为空的会员数量 |
| 畅享会员 | 有效会员卡中 `180 <= valid_days < 365` 的会员数量 |
| 畅享会员-单位会员 | 畅享会员中 `member.remark` 非空的会员数量 |
| 畅享会员-自然会员 | 畅享会员中 `member.remark` 为空的会员数量 |

说明：当前按会员卡有效天数识别尊享/畅享。后续如果卡类型表增加业务分类字段，应优先按卡类型分类统计。

### 登录与会员码展示数据

| 页面字段 | 统计规则 |
|---|---|
| 用户登录总量 | `xhbs_api_log` 中小程序登录接口成功请求总数 |
| 年用户登录量 | 当年小程序登录接口成功请求数 |
| 月用户登录量 | 当月小程序登录接口成功请求数 |
| 昨日用户登录量 | 昨日 00:00:00 至今日 00:00:00 前的小程序登录接口成功请求数 |
| 会员码展示总量 | 当前数据权限范围内会员调用会员码接口成功次数 |
| 年会员码展示量 | 当前数据权限范围内会员当年调用会员码接口成功次数 |
| 月会员码展示量 | 当前数据权限范围内会员当月调用会员码接口成功次数 |
| 昨日会员码展示量 | 当前数据权限范围内会员昨日调用会员码接口成功次数 |

口径说明：

- 小程序登录接口：`/api/mp/v1/auth/wechat-phone-login`，成功条件为 `response_code = 0`。
- 会员码接口：`/api/mp/v1/user/member-code`，成功条件为 `response_code = 0`。
- 会员码展示量通过 `xhbs_api_log.user_id = 'M' + member.id` 关联会员，再按会员所属部门纳入当前数据权限范围。
- 历史登录日志没有可靠的会员门店字段，因此用户登录量当前按全局成功登录次数统计；页面需要保留口径提示，避免误解为门店级登录量。

## 页面元素

| 区域 | 内容 |
|---|---|
| 顶部标题 | 会员数据看板 |
| 数据范围 | 展示当前统计范围，例如具体门店名或“可见部门 N 个” |
| 更新时间 | 展示 Redis 最后刷新时间 |
| 刷新按钮 | 手动触发统计刷新 |
| 会员指标卡 | 累计会员、借阅卡会员、普通会员、单位借阅卡会员、自然借阅卡会员、可见部门 |
| 会员卡结构表 | 尊享会员、畅享会员及单位/自然拆分 |
| 小程序使用数据 | 登录量和会员码展示量的总/年/月/昨日数据 |

## 异常处理

| 场景 | 处理方式 |
|---|---|
| 接口返回 401 | 跳转登录或提示重新登录 |
| 接口返回 500 | 页面提示接口异常，后端记录异常日志 |
| Redis 缓存为空 | 后端自动触发一次刷新 |
| 部分部门缓存缺失 | 返回 `missingDeptCount`，页面提示有部门暂无缓存 |
| 刷新锁被占用 | 返回提示“已有其他节点正在刷新，请稍后查看” |

## 验收标准

1. 不同数据权限账号登录首页，只能看到自己权限范围内的会员与会员码展示数据。
2. 门店账号只能看到本门店数据。
3. 经理账号能看到自己管辖部门及子部门合并数据。
4. 超级管理员能看到全部正常部门合并数据。
5. 页面刷新按钮可触发 Redis 统计刷新。
6. 多节点部署时，同一时间只有一个节点执行统计刷新。
7. 页面字段含义与本文件统计规则一致。
