# 后台会员卡退卡

## 基本信息

| 项 | 内容 |
|---|---|
| 接口 | `POST /member/card/{memberCardId}/refund` |
| 权限 | `member:card:refund` |
| 日志 | `@Log(title = "会员卡退卡", businessType = INSERT)` |
| 返回类型 | `AjaxResult` |

## 入参

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| memberCardId | path | long | 是 | 会员卡 ID |
| refundAmount | body | decimal | 是 | 退款金额 |
| refundType | body | string | 是 | 退款方式，如 `CASH` |
| reason | body | string | 否 | 退卡原因 |
| remark | body | string | 否 | 备注 |

## 完整请求报文

```http
POST /member/card/9001/refund HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <token>
Content-Type: application/json

{
  "refundAmount": 399.00,
  "refundType": "CASH",
  "reason": "7天内退卡",
  "remark": "前台现金退回"
}
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | 状态码 |
| msg | string | 提示信息 |
| data.refundOrderNo | string | 退卡订单号 |
| data.saleOrderNo | string | 原售卡订单号 |
| data.memberCardId | long | 会员卡 ID |
| data.refundAmount | decimal | 退款金额 |

## 完整响应报文

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "refundOrderNo": "REFUND202607050001",
    "saleOrderNo": "CARD202607050001",
    "memberCardId": 9001,
    "refundAmount": 399.00
  }
}
```

## 业务校验

- 会员卡必须存在。
- 退卡以付款时间 `paidAt` 为准，7 日内允许退卡，不以生效时间为准。
- 已退卡的会员卡不允许重复退卡。
- 成功后更新会员卡状态为 `3=已退卡`，写入 `refundOrderNo` 和 `refundedAt`。
- 成功后生成 `member_card_refund_order` 退卡订单。
- 成功后写入 `member_card_biz_log`，记录退卡前后状态、操作人、原因和订单号。
- 如果退的是当前生效卡，业务服务需要刷新会员卡队列，推动下一张待生效卡成为生效卡。

## 异常响应示例

```json
{
  "code": 500,
  "msg": "会员卡不在可退时间范围内"
}
```
