# 导出会员卡记录

## 基本信息

| 项 | 内容 |
|---|---|
| 接口 | `POST /member/card/export` |
| 权限 | `member:card:export` |
| 数据权限 | `@DataScope(deptAlias = "c")` |
| 返回类型 | Excel 文件流 |
| 导出范围 | 按当前查询条件导出全量数据，不受分页参数限制 |
| 默认排序 | `field(c.status, 1, 0, 3, 2), coalesce(c.paid_at, c.effective_at, c.created_at) desc, c.id desc` |

## 入参

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| memberId | form | long | 否 | 会员 ID |
| memberName | form | string | 否 | 会员姓名，模糊查询 |
| memberPhone | form | string | 否 | 手机号，模糊查询 |
| memberNo | form | string | 否 | 会员编号，模糊查询 |
| cardTypeId | form | int | 否 | 卡类型 ID |
| status | form | int | 否 | `0=待生效`，`1=生效中`，`2=已过期`，`3=已退款` |
| saleOrderNo | form | string | 否 | 售卡订单号，模糊查询 |
| refundOrderNo | form | string | 否 | 退款订单号，模糊查询 |
| deptId | form | long | 否 | 开卡门店 ID |
| createStaffName | form | string | 否 | 操作员工姓名，模糊查询 |
| beginPaidAt | form | string | 否 | 付款开始时间，格式 `yyyy-MM-dd HH:mm:ss` |
| endPaidAt | form | string | 否 | 付款结束时间，格式 `yyyy-MM-dd HH:mm:ss` |
| beginEffectiveAt | form | string | 否 | 生效开始时间，格式 `yyyy-MM-dd HH:mm:ss` |
| endEffectiveAt | form | string | 否 | 生效结束时间，格式 `yyyy-MM-dd HH:mm:ss` |
| beginExpiredAt | form | string | 否 | 到期开始时间，格式 `yyyy-MM-dd HH:mm:ss` |
| endExpiredAt | form | string | 否 | 到期结束时间，格式 `yyyy-MM-dd HH:mm:ss` |

## 完整请求报文

```http
POST /member/card/export HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <token>
Content-Type: application/x-www-form-urlencoded

memberNo=65000000001&status=0&beginPaidAt=2026-07-01%2000%3A00%3A00&endPaidAt=2026-07-31%2023%3A59%3A59
```

## 出参

接口返回 Excel 文件流，前端文件名规则：

```text
会员卡记录_查询条件_yyyyMMddHHmmss.xlsx
```

示例：

```text
会员卡记录_状态待生效_会员编号65000000001_20260707153021.xlsx
```

## 业务说明

- 导出用于对接外部 ERP，必须保留订单号、会员编号、卡类型、金额、时间、门店和操作员工等对账字段。
- 待生效卡也会导出，状态为“待生效”，“是否当前生效”为“否”，生效/到期时间按系统会员卡队列顺延后的预计时间展示。
- 已退款卡也会导出，状态为“已退款”，并携带退款订单号、退款时间。
- 导出查询条件与会员卡记录列表一致，但导出不传分页参数，返回符合条件的全量数据。
- 导出结果受登录人的数据权限限制，门店员工只能导出本门店范围内的数据，具备更大数据权限的账号按其权限范围导出。
