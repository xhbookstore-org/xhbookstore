## 维护规则

- 新增接口必须单独创建一个 `.md` 文档。
- 每个接口文档必须包含：基本信息、权限、入参、出参、完整请求报文、完整响应报文、业务说明。
- 涉及资金、会员卡状态、积分、库存的接口必须补充业务校验和异常响应。
- 前端按钮权限必须和后端 `@PreAuthorize` 保持一致。
- 列表接口必须写清默认排序、分页参数和数据权限。
