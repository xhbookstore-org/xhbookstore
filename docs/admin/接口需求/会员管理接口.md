# 会员管理接口

> 基础路径：`/member` | 认证方式：Session Cookie (SpringSecurity)

---

## 接口清单

| 编号 | 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|------|
| M-01 | GET | `/member/list` | 是 | 会员列表（分页+条件查询） |
| M-02 | GET | `/member/{id}` | 是 | 会员详情（含扩展表） |
| M-03 | POST | `/member` | 是 | 新增会员 |
| M-04 | PUT | `/member/{id}` | 是 | 编辑会员 |
| M-05 | DELETE | `/member/{id}` | 是 | 删除会员（逻辑删除） |
| M-06 | GET | `/member/generateCardNo` | 是 | 生成11位会员卡号 |
| M-07 | GET | `/member/cardTypes` | 是 | 卡类型列表 |
| M-08 | GET | `/member/{id}/points` | 是 | 会员积分流水列表 |
| M-09 | POST | `/member/{id}/points` | 是 | 添加积分 |
| M-10 | GET | `/member/export` | 是 | 导出会员数据 |

## M-01 会员列表

> **默认查询全部状态**（正常=0, 注销=1, 挂失=2），可通过 `status` 参数筛选。

```
GET /member/list?status=&name=&phone=&deptId=&beginTime=&endTime=&pageNo=1&pageSize=20
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | int | 否 | 会员状态：0=正常, 1=注销, 2=挂失。不传则查询全部 |
| name | string | 否 | 姓名（模糊匹配） |
| phone | string | 否 | 手机号（精确匹配） |
| deptId | long | 否 | 门店ID |
| beginTime | date | 否 | 注册开始日期 |
| endTime | date | 否 | 注册结束日期 |
| pageNo | int | 否 | 页码，默认1 |
| pageSize | int | 否 | 每页条数，默认20 |

### 返回字段

```json
{
  "code": 0,
  "msg": "操作成功",
  "rows": [
    {
      "id": 1,
      "cardNo": "65000000001",
      "name": "张三",
      "phone": "13800000001",
      "cardTypeName": "年卡",
      "deptName": "新华书店总店",
      "validDate": "2027-06-30",
      "borrowCountValid": 5,
      "currentPoints": 100,
      "remark": "某单位",
      "lastOperator": "admin",
      "status": 0,
      "source": "manual",
      "createdAt": "2026-06-30 10:00:00"
    }
  ],
  "total": 100
}
```

### 异常码

| code | 说明 |
|------|------|
| 500 | 查询失败 |

## M-03 新增会员

```
POST /member
Content-Type: application/json
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 姓名 |
| phone | string | 是 | 手机号（全局唯一） |
| cardTypeId | int | 是 | 卡类型ID |
| remark | string | 否 | 备注 |
| deptId | long | 否 | 门店ID（默认当前用户门店） |

### 后端自动生成

- `card_no`：11位卡号（门店编码+自增）
- `valid_date`：根据卡类型自动计算
- `join_date`：当天日期
- `source`：标记为 `manual`
- `business_staff_name`：当前操作人

### 返回字段

```json
{
  "code": 0,
  "msg": "操作成功"
}
```

### 异常码

| code | 说明 |
|------|------|
| 400 | 参数校验失败（必填字段为空/手机号格式错误） |
| 500 | 手机号已存在 / 新增失败 |

## M-04 编辑会员

```
PUT /member/{id}
Content-Type: application/json
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 否 | 姓名 |
| phone | string | 否 | 手机号 |
| cardTypeId | int | 否 | 卡类型ID（涉及有效期重算） |
| remark | string | 否 | 备注 |

### 业务规则

- 续费类型：有效期在原 valid_date 基础上叠加
- 新卡类型：有效期从当前日期重新计算
- 自动更新 last_operator 和 updated_at

### 异常码

| code | 说明 |
|------|------|
| 400 | 参数校验失败 |
| 500 | 手机号已被其他会员使用 / 更新失败 |

| code | 说明 |
|------|------|

## M-06 生成会员卡号

```
GET /member/generateCardNo?deptId=103
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| deptId | long | 否 | 门店ID（默认当前用户门店） |

### 生成规则

- 11位纯数字
- 前N位 = 门店 erp_dept_id
- 剩余位 = 门店内自增序列
- 全局排重

### 返回字段

```json
{ "code": 0, "msg": "ok", "data": "65000000005" }
```

## M-09 添加积分

```
POST /member/{id}/points
Content-Type: application/json
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| points | int | 是 | 积分值（正整数） |
| description | string | 否 | 积分事由 |

### 并发安全

悲观锁 + 事务：`SELECT member FOR UPDATE` → 记操作前后积分 → INSERT order → INSERT into_bill → UPDATE member → COMMIT

### 返回字段

```json
{ "code": 0, "msg": "操作成功" }
```

### 异常码

| code | 说明 |
|------|------|
| 400 | 积分值为空或非正数 |
| 500 | 会员不存在 / 积分增加失败 |

## M-10 导出会员

> **默认导出全部状态**，可通过 `status` 参数筛选。

```
GET /member/export?status=&name=&phone=&deptId=&beginTime=&endTime=
```

### 请求参数

同 M-01 查询参数（含 status，不含分页参数）

### 返回字段

导出 Excel 文件，包含列表全量字段 + 隐藏扩展字段

### 文件命名

`赤峰新华书店会员数据_导出日期.xlsx`
