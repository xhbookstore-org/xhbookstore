# 注销账号

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/account/cancel` |
| 认证 | 用户 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 对应代码 | `AccountController.cancel()` |

## 入参

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| reason | string | 否 | 注销原因 |
| confirmRead | boolean | 是 | 是否确认阅读注销须知，必须为 `true` |

## 完整请求报文

```http
POST /api/mp/v1/account/cancel HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <userToken>
Content-Type: application/json;charset=UTF-8
Idempotency-Key: account-cancel-1001-20260706153000

{
  "reason": "不再使用",
  "confirmRead": true
}
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| data.success | boolean | 是否注销成功 |
| data.memberId | long | 注销的会员 ID |
| data.cancelledAt | long | 注销时间，毫秒时间戳 |

## 完整响应报文

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "success": true,
    "memberId": 1001,
    "cancelledAt": 1783227000000
  },
  "requestId": "req_20260705_110060_001"
}
```

## 业务规则

- `confirmRead` 必须为 `true`，否则拒绝注销。
- 有未归还图书时不允许注销。
- 当前账号仍是有效员工身份时不允许注销。
- 注销为逻辑注销，会员状态置为注销/停用，不做物理删除。
- 该接口支持 `Idempotency-Key` 防重复提交。
