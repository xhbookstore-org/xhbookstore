# 通用安全与幂等规则

## 适用范围

本文档适用于小程序接口：`/api/mp/v1/**`。

## 认证规则

| 项目 | 规则 |
|---|---|
| Token 传递方式 | 只支持请求头：`Authorization: Bearer <accessToken>` |
| URL Token | 不支持 `?token=xxx` |
| 未登录 | 返回未认证错误 |
| Token 过期 | 调用刷新 Token 接口获取新 Token |

## 身份拦截规则

| 接口前缀 | 身份要求 |
|---|---|
| `/auth/**` | 不要求登录，具体接口按自身规则处理 |
| `/files/**` | 按文件接口自身规则处理 |
| `/user/**` | 必须是有效会员身份 |
| `/account/**` | 必须是有效会员身份 |
| `/staff/**` | 必须是有效员工身份 |

## 数据权限规则

| 场景 | 校验规则 |
|---|---|
| 用户查询借阅详情 `/user/borrows/{detailId}` | 明细所属 `memberId` 必须等于当前登录会员 ID |
| 用户查询积分详情 `/user/points-records/{pointsRecordId}` | 积分记录所属 `memberId` 必须等于当前登录会员 ID |
| 员工接口携带 `memberId` | 会员必须在当前员工可见部门数据范围内 |
| 员工接口携带 `detailId` | 借阅明细所属会员必须在当前员工可见部门数据范围内 |
| 员工接口携带 `pointsRecordId` | 积分记录所属会员必须在当前员工可见部门数据范围内 |

## 幂等规则

写接口支持请求头 `Idempotency-Key`，用于防止重复提交。

| 项目 | 规则 |
|---|---|
| 适用方法 | `POST`、`PUT` |
| 排除路径 | `/auth/**`、`/files/**` |
| 请求头 | `Idempotency-Key: <业务唯一键>` |
| 首次请求 | 写入处理中标记并正常执行业务 |
| 首次成功 | 缓存成功 JSON 响应 |
| 处理中重复提交 | 返回 `409`，提示请求正在处理中 |
| 成功后重复提交 | 直接返回首次成功响应 |
| 未传幂等键 | 不启用幂等缓存，按普通请求处理 |

## 完整请求示例

```http
POST /api/mp/v1/staff/members/1001/points/add HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <staffToken>
Content-Type: application/json;charset=UTF-8
Idempotency-Key: points-add-1001-20260706153000

{
  "reasonId": 1,
  "points": 10,
  "remark": "活动赠送"
}
```

## 异常响应示例

```json
{
  "code": 409,
  "message": "请求正在处理中，请勿重复提交",
  "data": null,
  "requestId": "req_20260706_153000_001"
}
```

## 环境规则

- `dev` 环境允许手机号 mock 登录，方便本地联调。
- `staging`、`prod` 环境不允许 11 位手机号 mock 兜底登录，必须走真实微信手机号授权流程。
