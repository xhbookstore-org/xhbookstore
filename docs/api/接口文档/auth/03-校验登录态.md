# 校验登录态

> **接口路径**: `GET /api/mp/v1/auth/session`
> **认证要求**: 是（Bearer Token）
> **Content-Type**: 无（GET 请求）
> **对应代码**: `AuthController.java`

## 接口说明

校验当前 `accessToken` 是否有效，返回完整的用户身份信息。适用于小程序启动时检查登录态，或页面权限校验。

---

## 请求参数

无请求体。

### 请求头

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Authorization | String | 是 | `Bearer <accessToken>` |

## curl 测试

```bash
curl -X GET http://localhost:8091/api/mp/v1/auth/session \
  -H "Authorization: Bearer <accessToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| data.valid | Boolean | 登录态是否有效 |
| data.isStaff | Boolean | 是否员工 |
| data.isMember | Boolean | 是否会员 |
| data.staffUserId | Long | 员工 user_id（isStaff=true 时有值） |
| data.userId | String | 用户标识（M/S 前缀 或 UUID） |

## 成功响应

### 会员+员工双身份

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": true,
    "isStaff": true,
    "isMember": true,
    "staffUserId": 8,
    "userId": "M3"
  },
  "requestId": "req_20260630120201"
}
```

### 纯会员

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": true,
    "isStaff": false,
    "isMember": true,
    "staffUserId": null,
    "userId": "M1"
  },
  "requestId": "req_20260630120202"
}
```

### 纯员工

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": true,
    "isStaff": true,
    "isMember": false,
    "staffUserId": 5,
    "userId": "S5"
  },
  "requestId": "req_20260630120203"
}
```

### Token 无效/未提供 → code 10004（由 JWT 过滤器拦截）

```json
{
  "code": 10004,
  "message": "无效的访问令牌",
  "data": null,
  "requestId": null
}
```

### Token 已过期 → code 10003

```json
{
  "code": 10003,
  "message": "登录已过期，请重新登录",
  "data": null,
  "requestId": null
}
```

## 业务逻辑

1. 请求经 `JwtAuthenticationFilter` 解析 Token，提取 claims 注入 `HttpServletRequest` 属性
2. 控制器读取 `userId`、`isStaff`、`isMember`、`staffUserId`
3. `userId` 为 null → `valid=false`
