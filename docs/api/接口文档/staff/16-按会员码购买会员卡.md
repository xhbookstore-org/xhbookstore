# 按会员码购买会员卡

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/staff/member-cards/buy` |
| 认证 | 员工 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 对应代码 | `StaffController.buyMemberCard()` |

## 接口说明

员工扫码后直接按 `memberCodeToken` 购买会员卡，不需要在路径中传 `memberId`。后端通过 token 定位会员。

本接口使用 `verifyToken` 校验二维码 token，不会立即消费 token。token 在有效期内可继续用于扫码预览或再次校验，直到服务端过期时间自然失效。

## 入参

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| memberCodeToken | body | string | 是 | 用户动态会员码 token |
| scanResult | body | string | 否 | 兼容旧字段；新业务统一使用 `memberCodeToken` |
| cardTypeId | body | int | 是 | 卡类型 ID |
| paidAmount | body | decimal | 是 | 实收金额 |
| paymentType | body | string | 否 | 支付方式，如 `STAFF_INPUT` |
| remark | body | string | 否 | 开卡备注 |

## 完整请求报文

```http
POST /api/mp/v1/staff/member-cards/buy HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <staffToken>
Content-Type: application/json;charset=UTF-8
Idempotency-Key: card-buy-token-20260707093000

{
```

## 业务规则

- 本接口由 `memberCodeToken` 定位会员。
- `memberCodeToken` 只校验不消费，按服务端有效期自然过期。
- 已有生效卡时，新卡待生效；当前生效卡过期后，下一张待生效卡顺延生效。
- 接口支持 `Idempotency-Key` 防重复提交。
