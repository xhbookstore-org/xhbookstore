# 开通续费会员卡

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/staff/members/{memberId}/activate-card` |
| 认证 | 员工 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 对应代码 | `StaffController.activateCard()` |

## 接口说明

员工在小程序端给指定会员购买会员卡。接口必须携带用户出示二维码后得到的 `memberCodeToken`，后端会校验 token 未过期、未被消费，并校验 token 解析出的会员与路径 `memberId` 一致，防止伪造会员身份。

## 入参

### 路径参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| memberId | long | 是 | 会员 ID，必须和 `memberCodeToken` 解析出的会员一致 |

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| memberCodeToken | string | 是 | 用户动态会员码 token，开卡时会被消费 |
| cardTypeId | int | 是 | 卡类型 ID |
| paidAmount | decimal | 是 | 员工录入的实收金额 |
| paymentType | string | 否 | 支付方式，如 `STAFF_INPUT`、`CASH` |
| remark | string | 否 | 备注 |

## 完整请求报文

```http
POST /api/mp/v1/staff/members/1001/activate-card HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <staffToken>
Content-Type: application/json;charset=UTF-8
Idempotency-Key: card-activate-1001-20260706153000

{
  "memberCodeToken": "MCT_20260705103000_8f7c6d",
  "cardTypeId": 1,
  "paidAmount": 399.00,
  "paymentType": "STAFF_INPUT",
  "remark": "门店开通年卡"
}
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | `0=成功` |
| message | string | 提示信息 |
| data.saleOrderNo | string | 售卡订单号 |
| data.memberCardId | long | 会员卡 ID |
| data.memberId | int | 会员 ID |
| data.memberNo | string | 会员编号 |
| data.cardTypeId | int | 卡类型 ID |
| data.cardTypeName | string | 卡类型名称 |
| data.paidAmount | decimal | 实收金额 |
| data.status | int | 会员卡状态 |
| data.effectiveAt | string | 生效时间；待生效卡可能为空 |
| data.expiredAt | string | 到期时间；待生效卡可能为空 |
| requestId | string | 请求追踪 ID |


## 业务规则

- 当前支付暂按员工录入金额处理，提交即视为支付成功。
- 成功后生成售卡订单 `member_card_order` 和会员卡 `member_card`。
- 如果会员当前没有生效卡，新卡直接生效；如果已有生效卡，新卡进入待生效队列。
- `memberCodeToken` 会被消费，避免重复提交、重复开卡。
- 该接口支持 `Idempotency-Key` 防重复提交。
