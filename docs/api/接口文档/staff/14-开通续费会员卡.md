# 开通续费会员卡

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/staff/members/{memberId}/activate-card` |
| 认证 | 员工 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 对应代码 | `StaffController.activateCard()` |

## 接口说明

员工扫码用户动态会员码后，给指定会员开卡或续卡。接口必须携带 `memberCodeToken`，后端校验 token 未过期、场景正确，并校验 token 解析出的会员与路径 `memberId` 一致。

本接口使用 `verifyToken` 校验二维码 token，不会立即消费 token。token 在有效期内可继续用于扫码预览或再次校验，直到服务端过期时间自然失效。

## 入参

### 路径参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| memberId | long | 是 | 会员 ID，必须和 `memberCodeToken` 解析出的会员一致 |

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| memberCodeToken | string | 是 | 用户动态会员码 token |
| cardTypeId | int | 是 | 卡类型 ID |
| paidAmount | decimal | 是 | 员工录入的实收金额 |
| paymentType | string | 否 | 支付方式，如 `STAFF_INPUT`、`CASH` |
| memberName | string | 否 | 会员姓名；有值时更新 `member.name`，售卡订单也使用更新后的姓名 |
| remark | string | 否 | 开卡备注，会同步到会员卡、售卡订单、会员列表备注展示 |

## 完整请求报文

```http
POST /api/mp/v1/staff/members/1001/activate-card HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <staffToken>
Content-Type: application/json;charset=UTF-8

{
  "memberCodeToken": "MCT_20260707093000_8f7c6d",
  "cardTypeId": 1,
  "paidAmount": 399.00,
  "paymentType": "STAFF_INPUT",

  "remark": "门店开通年卡"
}
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | `0=成功` |
| message | string | 提示信息 |
| data.saleOrderNo | string | 售卡订单号 |
| data.memberCardId | long | 会员卡 ID |
| data.memberId | int | 会员 ID |
| data.memberNo | string | 会员编号 |
| data.memberName | string | 会员姓名 |
| data.cardTypeId | int | 卡类型 ID |
| data.cardTypeName | string | 卡类型名称 |
| data.cardStatus | int | 会员卡状态，`1=生效中`、`0=待生效` |
| data.paidAmount | decimal | 实收金额 |
| data.effectiveAt | long | 生效时间毫秒时间戳 |
| data.expiredAt | long | 到期时间毫秒时间戳 |
| data.expectedEffectiveAt | long | 预计生效时间毫秒时间戳 |
| data.expectedExpiredAt | long | 预计到期时间毫秒时间戳 |
| data.remark | string | 开卡备注 |
| requestId | string | 请求追踪 ID |

## 业务规则

- 当前支付暂按员工录入金额处理，提交即视为支付成功。
- 成功后生成售卡订单 `member_card_order` 和会员卡 `member_card`。
- 如果会员当前没有生效卡，新卡直接生效；如果已有生效卡，新卡进入待生效队列。
- `memberCodeToken` 只校验不消费，按服务端有效期自然过期。
- 接口支持 `Idempotency-Key` 防重复提交。
