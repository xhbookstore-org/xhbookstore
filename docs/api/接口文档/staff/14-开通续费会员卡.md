# 开通/续费会员卡

> **接口路径**: `POST /api/mp/v1/staff/members/{memberId}/activate-card`
> **认证要求**: 是（员工 Bearer Token）
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java`

## 接口说明

员工为会员开通新卡或续费现有会员卡。
- **新卡（ACTIVATE）**：从当天起算有效期
- **续费（RENEW）**：在原到期日上叠加有效期（要求卡未过期）

---

## 请求参数

### 路径参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| memberId | Integer | 是 | 会员ID |

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cardTypeId | Integer | 是 | 卡类型ID（从 `/card-types` 接口获取） |
| remark | String | 否 | 备注 |

## 请求报文

```json
{
  "cardTypeId": 1,
  "remark": "门店开通年卡"
}
```

## 业务逻辑

### 核心流程

```
POST /members/{memberId}/activate-card
        │
        ├─ 1. 校验 cardTypeId → 查 card_type 表
        │
        ├─ 2. SELECT member FOR UPDATE（悲观锁，防并发重复开卡）
        │
        ├─ 3. 判断 is_renewal
        │      │
        │      ├─ 0（新卡）
        │      │    └─ 有效期 = 今天 + valid_days
        │      │    └─ operationType = "ACTIVATE"
        │      │
        │      └─ 1（续费）
        │           ├─ 原到期日 > 今天 → 正常续费
        │           │    └─ 有效期 = 原到期日 + valid_days
        │           │    └─ operationType = "RENEW"
        │           │
        │           └─ 原到期日 ≤ 今天 → ❌ 拒绝
        │                └─ "会员卡已过期，请选择「年卡」或「半年卡」开通新卡"
        │
        ├─ 4. UPDATE member SET card_type_id, valid_date, status=0
        │
        ├─ 5. INSERT member_card_log（变更前后完整记录）
        │
        └─ 6. 返回结果
```

### 有效期计算对照表

| 场景 | 旧卡状态 | 操作 | 有效期计算 | 示例 |
|------|----------|------|------------|------|
| 首次开卡 | 无 | ACTIVATE | 今天 + valid_days | 7/2 + 365 = 明年7/2 |
| 换卡 | 已过期 | ACTIVATE | 今天 + valid_days | 同上 |
| 续费 | 有效期内 | RENEW | 原到期日 + valid_days | 原到期7/2 + 365 = 后年7/1 |
| 过期续费 | 已过期 | ❌ 拒绝 | — | 提示选新卡 |

### 防并发

使用 `SELECT ... FOR UPDATE` 悲观锁，同一会员并发开卡时，后提交的事务等待前一个提交后才能读取最新数据，保证不会重复开卡或计算错误。

## 操作日志表

开卡/续费操作写入 `member_card_log` 表，记录变更前后的完整状态：

```sql
CREATE TABLE member_card_log (
    id                  BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    member_id           INT          NOT NULL                 COMMENT '会员ID',
    member_no           VARCHAR(50)  NOT NULL                 COMMENT '会员卡号',
    operation_type      VARCHAR(20)  NOT NULL                 COMMENT 'ACTIVATE=开卡 RENEW=续费',
    before_card_type_id INT          DEFAULT NULL             COMMENT '变更前卡类型ID',
    before_card_type    VARCHAR(50)  DEFAULT NULL             COMMENT '变更前卡类型名',
    before_valid_date   DATE         DEFAULT NULL             COMMENT '变更前到期日',
    after_card_type_id  INT          NOT NULL                 COMMENT '变更后卡类型ID',
    after_card_type     VARCHAR(50)  NOT NULL                 COMMENT '变更后卡类型名',
    after_valid_date    DATE         NOT NULL                 COMMENT '变更后到期日',
    operator_id         VARCHAR(50)  DEFAULT NULL             COMMENT '操作员工ID',
    operator_name       VARCHAR(50)  DEFAULT NULL             COMMENT '操作员工名',
    device              VARCHAR(20)  DEFAULT '小程序'          COMMENT '操作设备',
    remark              VARCHAR(255) DEFAULT NULL             COMMENT '备注',
    created_at          DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '操作时间',
    PRIMARY KEY (id),
    INDEX idx_member_id (member_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB COMMENT='会员开卡续费操作日志';
```

### 日志示例

```
id=1, member_id=10, operation_type=ACTIVATE
  before: NULL / NULL / NULL
  after:  年卡 / 2027-07-02
  remark: 门店开通年卡

id=2, member_id=10, operation_type=RENEW
  before: 年卡 / 2027-07-02
  after:  续费年卡 / 2028-07-01
  remark: 续费年卡
```
