# 消耗积分

> **接口路径**: `POST /api/mp/v1/staff/members/{memberId}/points/deduct`
> **认证要求**: 是 (Bearer Token)
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java` → `PointsServiceImpl.deductPoints()`

## 接口说明

扣减会员积分（书城币）。悲观锁+事务保证原子性，扣减后写入OUT类型订单和出账单。

## 路径参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| memberId | String | 是 | 会员ID |

## 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reasonId | String | 是 | 积分扣减事项ID |
| points | Integer | 是 | 扣减积分数（正整数） |
| remark | String | 否 | 备注 |

## 请求报文

```json
{
  "reasonId": "2",
  "points": 10,
  "remark": "兑换礼品扣减"
}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/staff/members/13/points/deduct \
  -H "Authorization: Bearer <staffToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{"reasonId":"2","points":10,"remark":"兑换扣减"}'
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.success | Boolean | 操作结果 |
| data.pointsRecordId | String | 积分流水单号（OUT开头） |
| data.beforePoints | Integer | 扣减前积分 |
| data.pointsDelta | Integer | 变动量（负值） |
| data.afterPoints | Integer | 扣减后积分 |
| data.operatedAt | Long | 操作时间 |

## 成功响应

```json
{
  "code": 0,
  "data": {
    "success": true,
    "pointsRecordId": "OUT20260702230000456789",
    "beforePoints": 320,
    "pointsDelta": -10,
    "afterPoints": 310,
    "operatedAt": 1782920000000
  }
}
```

## 错误响应

```json
{"code": 30001, "message": "积分不足，当前余额：5"}
{"code": 30003, "message": "无效的积分事项"}
{"code": 20001, "message": "会员不存在"}
```

## 业务逻辑（事务保证）

```
1. SELECT member FOR UPDATE                  ← 悲观锁
2. 校验 current_points >= points
3. 生成订单号 OUT + yyyyMMddHHmmss + 随机数
4. INSERT xhbs_points_order (amount=-points, orgin/after)
5. INSERT xhbs_points_user_out_bill_detail    ← 出账单
6. UPDATE member SET current_points = newValue
7. COMMIT                                     ← 事务提交
```

任意步骤失败 → 整体回滚，积分余额不变。
