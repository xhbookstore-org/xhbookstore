# 办理借阅

> 版本：V2.0 目标接口，当前代码待改造

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/staff/members/{memberId}/borrows` |
| 认证 | 员工 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 幂等 | 请求头 `Idempotency-Key` 必填 |
| 对应代码 | `StaffController.borrow()` |

## 数据边界

- 不接收 `bookId`，不查询或修改 `book_info`。
- 不校验、扣减或恢复库存，不写 book history。
- 一个 `books[]` 元素表示一册实体书，不接收合并数量 `borrowQty`。
- 每册图片放在当前 `books[]` 元素中，不接收订单顶层 `imageUrls`。

## 入参

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---:|---|
| `memberId` | path | long | 是 | 会员 ID |
| `books` | body | array | 是 | 逐册借阅明细，至少 1 条 |
| `books[].bookCode` | body | string | 是 | 员工手工填写的图书编号，按字符串保存 |
| `books[].bookName` | body | string | 是 | 员工手工填写的图书名称 |
| `books[].remark` | body | string | 否 | 当前这册书备注 |
| `books[].imageIds` | body | array | 否 | 当前这册书已上传的图片 ID，最多 3 个 |
| `remark` | body | string | 否 | 借阅订单备注 |

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| `data.orderNo` | string | 借阅单号 |
| `data.borrowOrderId` | long | 借阅单 ID |
| `data.totalBookCount` | int | 实际创建的逐册明细数 |
| `data.details[].borrowDetailId` | long | 逐册明细 ID |
| `data.details[].bookCode` | string | 图书编号快照 |
| `data.details[].bookName` | string | 图书名称快照 |
| `data.details[].imageCount` | int | 该册绑定图片数 |
| `data.points.status` | string | `SUCCESS/SKIPPED/FAILED_RETRYABLE` |
| `data.points.points` | int | 实际奖励积分，未发放时为 0 |

## 响应示例

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "orderNo": "BR202607160001",
    "borrowOrderId": 5001,
    "totalBookCount": 2,
    "details": [

      {"borrowDetailId": 3002, "bookCode": "0000123456", "bookName": "活着", "imageCount": 1}

    "points": {"status": "SUCCESS", "points": 20}
  }
}
```

## 业务规则

1. 员工必须具有有效身份和办理权限，会员必须通过借阅资格校验。
2. `bookCode`、`bookName` 去除首尾空格后不得为空；`bookCode` 保留前导 0。
3. 一个元素创建一条 `book_borrow_detail`，`borrow_qty` 固定为 1。
4. 同一编号可以出现多次，每次都代表一册并产生独立明细。
5. 每个 `imageId` 必须存在、由当前员工上传、未绑定其他业务；每册最多 3 张。
6. 订单、逐册明细、图片绑定和借阅日志在同一事务中完成。
7. 借阅成功后，后端用 `BORROW_BOOK`、借阅单号和明细数调用积分规则服务；最终积分由 `xhbs_points_rule` 计算。
8. 同一 `Idempotency-Key` 和同一 `BORROW_BOOK:{orderNo}` 都不得重复创建数据或重复发积分。
9. 积分规则未命中或处理失败不撤销已成功的借阅单，需保留可查询状态并安全重试。
