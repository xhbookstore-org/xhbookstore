# 办理借阅

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/staff/members/{memberId}/borrows` |
| 认证 | 员工 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 对应代码 | `StaffController.borrow()` |

## 入参

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| memberId | path | long | 是 | 会员 ID |
| books | body | array | 是 | 借阅图书 |
| books[].bookId | body | long | 是 | 图书 ID |
| books[].bookName | body | string | 否 | 图书名称 |
| books[].borrowQty | body | int | 是 | 借阅数量 |
| remark | body | string | 否 | 备注 |
| imageUrls | body | array | 否 | 附件图片 URL |

## 完整请求报文

```http
POST /api/mp/v1/staff/members/1001/borrows HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <staffToken>
Content-Type: application/json;charset=UTF-8
Idempotency-Key: borrow-1001-20260706153000

{
  "books": [
    {
      "bookId": 2001,
      "bookName": "活着",
      "borrowQty": 2
    }
  ],
  "remark": "门店借阅",
  "imageUrls": [
    "https://cos.example.com/bookstore/1001/book.jpg"
  ]
}
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| data.orderNo | string | 借阅单号 |
| data.borrowOrderId | long | 借阅单 ID |
| data.totalBookCount | int | 借阅总数量 |

## 完整响应报文

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "orderNo": "BR202607050001",
    "borrowOrderId": 5001,
    "totalBookCount": 2
  },
  "requestId": "req_20260705_112060_001"
}
```

## 业务规则

- 员工必须有有效员工身份。
- 员工可以为任意门店会员办理借阅，会员资格和有效借阅卡仍需通过业务校验。
- `borrowQty` 必须大于 `0`。
- 批量借书在写入订单和扣减库存前完整校验所有图书、数量、会员额度及库存。
- 任意一本图书校验或库存扣减失败时抛出业务异常，整个事务回滚，已处理图书不会留下库存或借阅明细变化。
- 库存扣减使用“当前可借数量充足”条件的原子 SQL，并校验受影响行数，防止并发超借。
- 该接口支持 `Idempotency-Key` 防重复提交。
