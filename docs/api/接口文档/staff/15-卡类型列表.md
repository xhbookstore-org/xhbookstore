# 卡类型列表

> **接口路径**: `GET /api/mp/v1/staff/card-types`
> **认证要求**: 是（Bearer Token）
> **Content-Type**: 无（GET 请求）
> **对应代码**: `StaffController.java`

## 接口说明

获取当前可用的会员卡类型列表，用于开卡/续费时选择。数据源 `util_card_type` 表，筛选 `is_del=0` 且 `status=0`。

---

## 请求参数

无。

## curl 测试

```bash
curl http://localhost:8091/api/mp/v1/staff/card-types \
  -H "Authorization: Bearer <staffToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.list | Array | 卡类型列表 |
| data.list[].id | Integer | 卡类型ID（用于 activate-card 的 cardTypeId 参数） |
| data.list[].typeName | String | 名称（普通会员/年卡/半年卡/续费年卡/续费半年卡） |
| data.list[].price | BigDecimal | 售价（元），0=免费 |
| data.list[].validDays | Integer | 有效天数 |
| data.list[].borrowLimit | Integer | 最大同时借阅数 |
| data.list[].discount | BigDecimal | 购书折扣（1.00=无折扣） |
| data.list[].isRenewal | Boolean | 是否续费类型 |
| data.list[].description | String | 卡类型说明 |
| data.list[].sort | Integer | 排序 |
| data.list[].status | Integer | 状态（0=正常） |

## 业务逻辑

- 查询 `util_card_type` 表：`WHERE is_del=0 AND status=0 ORDER BY sort`
- `isRenewal` 已从 DB 的 `0/1` 转为 `true/false`
- `validDays`/`borrowLimit`/`discount`/`price` 直接从表字段读取，不再硬编码
