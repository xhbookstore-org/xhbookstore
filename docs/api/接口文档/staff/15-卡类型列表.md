# 卡类型列表

> **接口路径**: `GET /api/mp/v1/staff/card-types`
> **认证要求**: 是（Bearer Token）
> **Content-Type**: 无（GET 请求）
> **对应代码**: `StaffController.java`

## 接口说明

获取当前可用的会员卡类型列表，用于开卡/续费时选择。只返回 `status=0`（正常启用）的类型。

---

## 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 无 | — | — | 无需请求参数 |

## curl 测试

```bash
curl http://localhost:8091/api/mp/v1/staff/card-types \
  -H "Authorization: Bearer <staffToken>"
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| data.list | Array | 卡类型列表 |
| data.list[].id | Integer | 卡类型ID |
| data.list[].typeName | String | 名称（年卡/半年卡/续费年卡/续费半年卡） |
| data.list[].validDays | Integer | 有效天数（365/180） |
| data.list[].isRenewal | Boolean | 是否续费类型（true=续费/false=新卡） |
| data.list[].status | Integer | 状态（0=正常） |

## 成功响应

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "list": [
      {"id": 1, "typeName": "年卡",       "validDays": 365, "isRenewal": false, "status": 0},
      {"id": 2, "typeName": "半年卡",     "validDays": 180, "isRenewal": false, "status": 0},
      {"id": 3, "typeName": "续费年卡",   "validDays": 365, "isRenewal": true,  "status": 0},
      {"id": 4, "typeName": "续费半年卡", "validDays": 180, "isRenewal": true,  "status": 0}
    ]
  },
  "requestId": null
}
```

## 业务逻辑

- 查询 `card_type` 表，仅返回 `status=0` 的启用类型
- `isRenewal` 字段已从数据库 `0/1` 转换为 `true/false`，前端可直接用于 UI 判断
- 卡类型固定 4 种：年卡、半年卡、续费年卡、续费半年卡
