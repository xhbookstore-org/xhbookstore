# 办理还书（单册四态）

| 项目 | 说明 |
| --- | --- |
| 接口 | `POST /api/mp/v1/staff/borrow-returns` |
| 鉴权 | 员工 `Bearer accessToken` |
| 幂等 | 建议请求头传 `Idempotency-Key`，值按明细和本次操作唯一生成 |

## 请求体

一次请求只处理一册借阅明细；旧字段 `returnItems`、`borrowDetailIds`、`returnQty`、`returnType` 均不再支持。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `borrowDetailId` | long | 是 | 借阅明细 ID；仅可处理仍在借的一册 |
| `returnCondition` | string | 是 | `intact`、`slight_wear`、`damaged`、`purchase` 之一 |
| `points` | int | 条件必填 | `damaged`、`purchase` 时必填，非负整数，允许 `0`；其它状态忽略该字段 |
| `remark` | string | 否 | 店员补充备注；服务端会自动写入状态和积分金额摘要 |

| 产品文案 | `returnCondition` | 兼容落库 `returnType` | `points` |
| --- | --- | ---: | --- |
| 无损 | `intact` | 1 | 不需要 |
| 轻微磨损 | `slight_wear` | 2 | 不需要 |
| 破损 | `damaged` | 3 | 必填 |
| 购买 | `purchase` | 4 | 必填 |

`purchase` 通过本接口完成，不再调用借转购接口。它会将借阅明细标记为购买；其它三个状态将明细标记为已归还。

```json
{
  "borrowDetailId": 3001,
  "returnCondition": "damaged",
  "points": 0,
  "remark": "封面破损"
}
```

## 积分与落库

- `points` 同时写入 `book_return_detail.points` 和 `remark` 摘要。
- `damaged`、`purchase` 按已启用的 `PURCHASE_BOOK` 规则自动处理积分；规则方向决定增加或扣减，`PER_YUAN` 规则按员工录入金额计算并支持会员日倍数。
- `points=0` 合法，不产生积分流水。
- 还书明细、借阅明细状态、操作日志和积分流水在同一事务内处理；任一步失败会整体回滚。
