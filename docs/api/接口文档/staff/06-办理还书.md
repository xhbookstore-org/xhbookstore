# 办理还书

> **接口路径**: `POST /api/mp/v1/staff/borrow-returns`
> **认证要求**: 是（Bearer Token）
> **Content-Type**: application/json;charset=UTF-8
> **对应代码**: `StaffController.java`

## 接口说明

提交借阅明细ID数组，服务端自动查询对应明细完成归还。支持多本同时还书。

## 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| borrowDetailIds | Array\<Long\> | 是 | 借阅明细ID列表，如 [101,102,103] |

## 请求报文

```json
{
  "borrowDetailIds": [101, 102, 103]
}
```

## curl 测试

```bash
curl -X POST http://localhost:8091/api/mp/v1/staff/borrow-returns \
  -H "Authorization: Bearer <staffToken>" \
  -H "Content-Type: application/json;charset=UTF-8" \
  -d '{"borrowDetailIds":[101,102]}'
```

## 响应报文

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "returnedCount": 2
  }
}
```

## 错误响应

```json
{"code": 400, "message": "请选择要还的图书"}
{"code": 404, "message": "借阅明细101不存在"}
{"code": 40004, "message": "不允许办理还书"}
```

## 业务逻辑

1. 前端只提交 `borrowDetailIds`，不需要提交借书单号
2. 服务端根据 detailId 查借阅明细 → 获取借书单号 → 执行归还
3. 后端内部保留"部分归还"处理能力，但前端列表不暴露该状态
4. 已归还的明细不可重复归还
