# 上传借阅明细图片

> 版本：V2.0 目标接口，当前代码待改造

## 基本信息

| 项 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/files/book-attachment-images` |
| 认证 | 员工 Bearer Token |
| Content-Type | `multipart/form-data` |
| 对应代码 | `FileController.uploadBookImage()` |

本期该接口只服务于借阅明细图片，不提供 `book_info` 封面上传和 `bookId` 关联。

## 两种上传阶段

1. 借阅单提交前：不传 `borrowDetailId`，服务端返回暂存 `imageId`；办理借阅时通过 `books[].imageIds` 绑定到指定逐册明细。
2. 借阅单创建后：传 `borrowDetailId`，服务端校验数据权限和现有图片数后直接绑定。

## 入参

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `file` | file | 是 | 单张图片，最大 2MB，仅 JPEG/PNG |
| `borrowDetailId` | long | 否 | 已创建明细时传入 |
| `imageType` | int | 否 | 1借书/2归还/3损坏或其他凭证，默认 1 |

`memberId`、`borrowOrderId`、`borrowOrderNo` 不作为权限依据；已绑定明细时均由服务端根据 `borrowDetailId` 反查。

## 请求示例

```http
POST /api/mp/v1/files/book-attachment-images HTTP/1.1
Authorization: Bearer <staffToken>
Content-Type: multipart/form-data; boundary=----Boundary

------Boundary
Content-Disposition: form-data; name="borrowDetailId"

3001
------Boundary
Content-Disposition: form-data; name="imageType"

1
------Boundary
Content-Disposition: form-data; name="file"; filename="book.jpg"
Content-Type: image/jpeg

<binary>
------Boundary--
```

## 出参

| 字段 | 说明 |
|---|---|
| `data.imageId` | 图片业务 ID |
| `data.url` | 当前可访问原图地址 |
| `data.thumbUrl` | 缩略图地址 |
| `data.size` | 文件字节数 |
| `data.format` | `jpg/png` |
| `data.width/height` | 宽高 |
| `data.bindStatus` | `TEMP/BOUND` |
| `data.borrowDetailId` | 已绑定时返回明细 ID |

## 业务与安全规则

1. 只允许有效员工上传；已绑定明细必须通过门店和角色数据权限校验。
2. 每个 `borrow_detail_id` 的有效图片总数最多 3 张，不按 `imageType` 分别计数。
3. 服务端必须在事务/锁内重新统计图片数，并发上传第 4 张也必须只成功一个。
4. 不信任客户端 `Content-Type`，同时校验扩展名、JPEG/PNG 文件签名和图片解码结果。
5. 单文件最大 2MB，宽高和总像素受服务端限制。
6. 暂存图片带有上传人和失效时间，只能被同一员工的后续借阅请求绑定；过期未绑定对象由清理任务删除。
7. 对象存储上传成功但数据库写入失败时，必须执行补偿清理。
8. 日志记录调用身份、明细 ID/暂存 ID、对象 Key、大小、成功或失败原因，不记录访问令牌和密钥。
