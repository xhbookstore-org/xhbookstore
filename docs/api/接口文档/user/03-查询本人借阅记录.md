# 查询本人借阅记录

## 基本信息

| 项目 | 内容 |
|---|---|
| 接口 | `GET /api/mp/v1/user/borrows` |
| 认证 | 用户 Bearer Token |
| 对应代码 | `UserController.myBorrows()` |

## 入参

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| pageNo | query | int | 否 | 页码，默认 1，必须大于 0 |
| pageSize | query | int | 否 | 每页条数，默认 20，范围 1-100 |

## 完整请求报文

```http
GET /api/mp/v1/user/borrows?pageNo=1&pageSize=20 HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <userToken>
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| data.memberDisplay | string | 脱敏手机号 |
| data.yearBorrowCount | int | 本人借阅单数量 |
| data.currentBorrowingCount | int | 当前未还/未转购的图书总数量 |
| data.page.list | array | 借阅明细列表，每本书一条记录 |
| data.page.list[].detailId | long | 借阅明细 ID，进入详情时传这个值 |
| data.page.list[].borrowDetailId | long | 借阅明细 ID，兼容字段 |
| data.page.list[].orderNo | string | 借阅单号 |
| data.page.list[].bookCode | string | 借阅时录入的图书编号 |
| data.page.list[].bookName | string | 图书名称 |
| data.page.list[].borrowStatus | int | 明细借阅状态 |
| data.page.list[].borrowQty | int | 固定为 1 |
| data.page.list[].returnedQty | int | 本书已还数量 |
| data.page.list[].purchaseQty | int | 本书借转购数量 |
| data.page.list[].remainingQty | int | 本书当前可还数量 |
| data.page.list[].purchaseOrderNo | string | 借转购订单号，没有则为空 |
| data.page.list[].borrowTime | long | 借出时间，毫秒时间戳 |
| data.page.list[].returnAllTime | long | 本书全部归还/处理完成时间，未完成为空 |
| data.page.list[].expectedReturnTime | long | 借阅单预计归还时间，毫秒时间戳 |
| data.page.list[].remark | string | 备注 |
| data.page.pageNo | int | 当前页码 |
| data.page.pageSize | int | 每页条数 |
| data.page.total | long | 符合条件的借阅明细总数 |
| data.page.hasMore | boolean | 是否还有下一页 |

## 业务规则

- 会员 ID 从登录 Token 中解析，只返回当前登录会员本人的借阅明细。
- 服务端直接按逐册借阅明细进行 SQL 分页，同一张借阅单下多册书会返回多条记录。
- 总数、当前未处理数量和借阅单数量均由 SQL 聚合查询，不加载全量记录到内存计算。
- 前端进入详情时使用 `detailId` 调用 `/api/mp/v1/user/borrows/{detailId}`。
