# 生成动态会员码

## 基本信息

| 项 | 内容 |
|---|---|
| 接口 | `POST /api/mp/v1/user/member-code` |
| 认证 | 用户 Bearer Token |
| Content-Type | `application/json;charset=UTF-8` |
| 对应代码 | `UserController.generateMemberCode()` |

## 接口说明

用户在小程序端出示会员二维码前调用本接口。后端生成短有效期的 `memberCodeToken`，前端将 `codeContent` 编码为二维码，员工扫码后用该 token 识别会员并办理开卡。

## 入参

请求体为空对象。

| 字段 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 无 | body | object | 否 | 用户身份从 Token 解析 |

## 完整请求报文

```http
POST /api/mp/v1/user/member-code HTTP/1.1
Host: 152.136.127.168
Authorization: Bearer <userToken>
Content-Type: application/json;charset=UTF-8

{}
```

## 出参

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | `0=成功` |
| message | string | 提示信息 |
| data.memberNo | string | 会员编号 |
| data.memberCodeToken | string | 动态会员码 token，二维码实际内容 |
| data.codeContent | string | 二维码内容，格式 `MCODE:{memberCodeToken}` |
| data.expiresAt | long | 过期时间，毫秒时间戳 |
| data.ttlSeconds | int | 有效秒数 |
| requestId | string | 请求追踪 ID |

## 完整响应报文

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "memberNo": "65000000001",
    "memberCodeToken": "MCT_20260705103000_8f7c6d",
    "codeContent": "MCODE:MCT_20260705103000_8f7c6d",
    "expiresAt": 1783228230000,
    "ttlSeconds": 60
  },
  "requestId": "req_20260705_103000_001"
}
```

## 异常响应

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null,
  "requestId": "req_20260705_103000_002"
}
```

## 业务说明

- `memberCodeToken` 是防刷、防伪造的核心字段，不能由前端自己拼接。
- token 当前有效期为 60 秒，过期后员工端扫码会失败。
- 开卡时员工端必须提交同一个 token，后端会校验 token 对应会员和使用场景。
