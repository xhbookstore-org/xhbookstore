## 通用安全规则

- 登录后接口只支持请求头传 Token：`Authorization: Bearer <accessToken>`。
- 不再支持通过 URL 参数传 `?token=xxx`，避免 Token 泄露到日志、浏览器历史或代理链路。
- `/user/**`、`/account/**` 必须是有效会员身份。
- `/staff/**` 必须是有效员工身份。
- 涉及 `memberId`、`detailId`、`pointsRecordId` 的员工端接口，会按员工数据权限校验可见范围；用户端详情接口只能查询当前登录会员本人的数据。
- 写接口支持可选请求头 `Idempotency-Key`，用于防重复提交。首次成功响应会缓存；处理中重复请求返回冲突提示；成功后重复请求返回首次成功结果。

## 当前边界

- 用户端只生成动态会员码，不直接办理开卡和退卡。
- 员工端扫码开卡统一使用 `memberCodeToken` 防刷、防伪造。
- 员工端支付暂按员工录入金额处理，提交即视为支付成功。
- 退卡只在 admin 后台模块办理，小程序端不提供退卡接口。
- `dev` 环境允许模拟登录；`staging`、`prod` 环境不允许 11 位手机号 mock 兜底登录。
