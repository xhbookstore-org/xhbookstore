## 当前规则摘要

1. 小程序用户端只负责展示动态会员码，不直接办理退卡。
2. 员工端通过扫码拿到 `memberCodeToken` 后调用后端解析，后端校验 token 有效性。
3. 开卡支付当前为员工录入金额，提交即视为支付成功。
4. 退卡业务统一放在 admin 后台，API 小程序端不提供退卡入口。
5. 借阅、还书、借转购需要保证订单、明细、库存和日志闭环。

## 通用接口约定

- `Content-Type`: `application/json;charset=UTF-8`
- 认证方式：`Authorization: Bearer <accessToken>`
- 统一响应：`{"code":0,"message":"操作成功","data":{},"requestId":"..."}`
- 分页响应：`{"list":[],"pageNo":1,"pageSize":20,"total":100,"hasMore":true}`
