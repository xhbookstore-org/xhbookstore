# C01 — 登录弹层

> 对应代码: `AuthController.java` | 基础路径: `/api/mp/v1/auth`

微信小程序登录弹层，用户通过微信授权手机号完成登录认证。

---

## 关联接口

| 接口 | 方法 | 路径 | 认证 |
|------|------|------|------|
| 微信手机号登录 | POST | `/api/mp/v1/auth/wechat-phone-login` | 否 |
| 刷新Token | POST | `/api/mp/v1/auth/refresh-token` | 否 |
| 校验登录态 | GET | `/api/mp/v1/auth/session` | 是 |
| 退出登录 | POST | `/api/mp/v1/auth/logout` | 是 |

---

## 1. 微信手机号登录

```
POST /api/mp/v1/auth/wechat-phone-login
Content-Type: application/json;charset=UTF-8
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 是 | 微信手机号授权组件返回的凭证 |

### 请求报文

```json
{
  "code": "0a3f4b2c1d5e6f7g8h9i0j1k2l3m4n5o"
}
```

### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 0=成功 |
| message | String | 操作成功 |
| data.accessToken | String | 访问令牌（2小时有效） |
| data.refreshToken | String | 刷新令牌（30天有效） |
| data.expiresIn | Long | accessToken 有效期（秒） |
| data.isStaff | Boolean | 是否员工（在 sys_user 表中有记录） |
| data.staffUserId | Long | 员工 user_id（isStaff=true 时有值，否则 null） |
| data.isMember | Boolean | 是否会员（在 member 表中有记录） |
| data.userId | String | 用户标识：M{memberId} / S{staffUserId} / UUID |
| data.memberId | Integer | 会员ID（isMember=true 时有值，否则 null） |

### 响应报文（纯会员登录）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiJ9...",

```

## 3. 校验登录态

```
GET /api/mp/v1/auth/session
Authorization: Bearer <accessToken>
```

### 响应报文（已登录）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": true,
    "isStaff": false,
    "userId": "1"
  },
  "requestId": null
}
```

### 响应报文（未登录）

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "valid": false
  },
  "requestId": null
}
```

    "success": true
  },
  "requestId": null

## 业务逻辑

1. **双表查询**: 登录时同时查询 `member` 表和 `sys_user` 表，分别确定会员身份和员工身份。
2. **身份判断**: `isMember`=在 member 表有记录；`isStaff`=在 sys_user 表有记录且 status 正常。一个人可以同时是会员和员工。
3. **userId 规则**: 会员→`M{memberId}`，员工→`S{staffUserId}`，访客→UUID。
4. **双Token机制**: accessToken 短期（2h），refreshToken 长期（30d）。
5. **JWT 结构**: 包含 userId、isMember、isStaff、phone、memberId、staffUserId、type。
6. **登录限流**: 默认 5次/分钟/IP，dev 环境放宽至 100次/分钟。
