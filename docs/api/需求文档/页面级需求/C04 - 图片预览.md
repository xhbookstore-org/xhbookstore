# C04 — 借阅图片上传与预览

> 版本：V2.0 目标需求
>
> 对应接口：`POST /api/mp/v1/files/book-attachment-images`
> 详细报文：[files/01-上传图书附件图片](../../接口文档/files/01-上传图书附件图片.md)

## 1. 使用场景

- 办理借阅时，在每册图书卡片内拍照或选择图片。
- 借阅详情中展示当前逐册明细的缩略图，点击查看大图。
- 图片只关联借阅明细，不关联 `book_info`、`bookId` 或订单顶层。

## 2. 组件行为

1. 每条借阅明细最多显示 3 个图片位。
2. 已有 3 张时隐藏或禁用“继续上传”。
3. 上传中显示进度；失败可单张重试，不影响其他图片。
4. 借阅单提交前，图片以暂存 `imageId` 保存；提交时放入对应 `books[].imageIds`。
5. 删除前二次确认；已绑定图片的删除受员工权限和数据范围限制。

## 3. 文件规则

| 项 | 规则 |
|---|---|
| 格式 | JPEG、PNG |
| 单文件大小 | 最大 2MB |
| 数量 | 每个 `borrowDetailId` 有效图片总计最多 3 张 |
| 校验 | 扩展名、文件签名、图片解码、宽高和总像素 |
| 预览 | 列表用缩略图，大图使用受权限控制的访问地址 |

3 张上限是借阅明细的总上限，不按“借书/归还/损坏”分类各自计数。

## 4. 安全与异常

- 前端的数量限制不代替服务端并发约束。
- 只允许员工上传，服务端根据借阅明细反查会员、订单和门店权限。
- 暂存图片过期未绑定时自动清理。
- 对象存储成功但数据库失败时必须补偿清理，避免孤儿文件。

## 5. 验收标准

1. 每册最多 3 张，并发上传也不能超限。
2. 每张图片最终都能追溯到唯一 `borrow_detail_id`。
3. 页面和接口不出现 `bookId` 图书封面上传。
4. 缩略图可正常打开大图，无权用户不能通过直接 URL 绕过权限。
