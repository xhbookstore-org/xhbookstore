# E08 — 办理借阅页

> 版本：V2.0 目标需求
>
> 对应接口：`POST /api/mp/v1/staff/members/{memberId}/borrows`
> 详细报文：[staff/08-办理借阅](../../接口文档/staff/08-办理借阅.md)

## 1. 页面目标

员工扫码确认会员后，逐册手工录入图书编号、图书名称和图片，提交一张借阅单。

本页不提供图书主数据选择、搜索、导入、库存和数量合并输入。

## 2. 页面结构

### 2.1 会员概要

- 姓名、脱敏手机、会员编号。
- 当前会员卡、有效期和可借状态。
- 不可借时展示明确原因并禁用提交。

### 2.2 逐册录入卡片

每个卡片表示一册书：

| 字段 | 必填 | 说明 |
|---|---:|---|
| 图书编号 | 是 | 手工输入，保留前导 0 |
| 图书名称 | 是 | 手工输入 |
| 备注 | 否 | 当前这册书备注 |
| 图片 | 否 | 0~3 张，支持拍照/选择、删除和大图预览 |

操作：“添加一册”、“复制此册”、“删除此册”。借同一本书两册时显示两张卡片，不把数量填为 2。

### 2.3 订单备注与提交

- 可填写整张借阅单备注。
- 页面实时显示“共 N 册”，N 为录入卡片数。
- 提交中禁止重复点击，请求携带 `Idempotency-Key`。

## 3. 提交数据

```json
{
  "books": [
    {
      "bookCode": "0000123456",
      "bookName": "活着",
      "remark": "第一册",
      "imageIds": ["IMG001", "IMG002"]
    },
    {
      "bookCode": "0000123456",
      "bookName": "活着",
      "remark": "第二册",
      "imageIds": []
    }
  ],
  "remark": "门店借阅"
}
```

不提交 `bookId`、`borrowQty`、顶层 `imageUrls` 或任何库存字段。

## 4. 积分展示

- 借阅提交成功后，页面展示后端返回的实际奖励积分。
- 页面不写死“每册 10 分”，可在提交前通过规则预览接口展示“预计获得”。
- 积分规则未命中时显示“本次未发放积分”；积分异常不得将已成功借阅误提示为失败。

## 5. 验收标准

1. 图书编号和名称均为必填手工输入项。
2. 同一本书两册显示两个逐册卡片，后端创建两条明细。
3. 每个卡片最多 3 张图片，第 4 张在前端不可继续选择，后端也必须拒绝。
4. 页面不出现图书 ID、图书库存、图书导入和数量输入。
5. 积分以后端 `xhbs_points_rule` 计算结果为准，页面不能指定或修改最终奖励。
